# 互動設計分工（4/9 會議版）

> **目標**：六核心功能到齊了，但互動深度淺——每個功能只有一層反應，沒有複合場景或連續對話。這份文件定義「偵測到之後 → Go2 做什麼」的擴充方向。

## 你不需要的東西

你**不需要** Jetson、Go2、Intel RealSense D435 深度攝影機。

所有開發都可以在你自己的電腦上完成：
- **前端 UI**：Mock Server 模擬所有後端資料（見 [README.md](README.md)）
- **互動規則**：寫 JSON/YAML 設定檔，Roy 整合到 Jetson
- **測試手勢/姿勢**：用你自己的鏡頭，高度 ~30cm 模擬 Go2 視角

## Go2 可用動作 API 參考

### 常用動作（推薦使用）

| api_id | 動作名稱 | 說明 | 適合場景 |
|:------:|----------|------|----------|
| **1003** | StopMove | 緊急停止 | stop 手勢、障礙物、跌倒 |
| **1004** | StandUp | 從坐下站起來 | 語音指令「站起來」 |
| **1009** | Sit | 坐下 | 語音指令「坐下」、休息 |
| **1016** | Hello | 招手/打招呼動畫 | 人臉辨識到人、語音打招呼 |
| **1020** | Content | 開心/滿足姿態 | thumbs_up、ok 手勢、正面回饋 |
| **1017** | Stretch | 伸懶腰 | 無聊、等待、可愛動作 |
| **1022** | Dance1 | 跳舞動作 1 | 開心、慶祝 |
| **1023** | Dance2 | 跳舞動作 2 | 開心、慶祝 |
| **1021** | Wallow | 打滾 | 撒嬌、可愛動作 |
| **1029** | Scrape | 刨地動作 | 可愛、模仿狗行為 |
| **1036** | FingerHeart | 比愛心（前腳） | 可愛互動 |
| **1033** | WiggleHips | 搖屁股 | 開心、撒嬌 |
| **1032** | FrontPounce | 前撲 | 興奮、互動 |

### 移動動作

| api_id | 動作名稱 | 說明 | 備註 |
|:------:|----------|------|------|
| **1008** | Move | 移動命令 | 需要搭配方向/速度參數 |
| **1002** | BalanceStand | 平衡站立 | 站穩不動 |
| **1005** | StandDown | 降低身高 | 蹲下一點 |
| **1006** | RecoveryStand | 摔倒後自己站起來 | 跌倒恢復 |

### 🚫 禁止使用（危險）

| api_id | 動作名稱 | 禁止原因 |
|:------:|----------|----------|
| 1030 | FrontFlip | 前空翻，摔倒風險極高 |
| 1031 | FrontJump | 前跳，摔倒風險極高 |
| 1301 | Handstand | 倒立，不穩定 |

### 語音回應（TTS）

除了動作，Go2 還可以**說話**。任何場景都可以搭配 TTS 語音回應：

```
觸發 → Go2 動作 + TTS 語音
例：thumbs_up → Content(1020) + TTS「謝謝你！」
例：fallen → StopMove(1003) + TTS「偵測到跌倒！請注意安全」
```

## 目前各功能的互動現況 vs 要擴充什麼

### 🖐 手勢辨識（黃旭）

**目前有的（只有 3 個映射）**：

| 手勢 | Go2 動作 | TTS | 冷卻 |
|------|---------|-----|:----:|
| stop | StopMove(1003) | — | 無 |
| ok | Content(1020) | — | 3s |
| thumbs_up | Content(1020) | 「收到！」 | 3s |

**你要思考的**：

1. **thumbs_up 之後 Go2 應該做什麼？** 只有 Content 太無聊了。要不要加跳舞？搖屁股？
2. **wave（揮手）應該觸發什麼？** 目前什麼都沒做。打招呼？揮手回應？
3. **point（指向）應該觸發什麼？** 目前什麼都沒做。可以讓 Go2 看向手指方向？
4. **要不要新增手勢？** MediaPipe Gesture Recognizer 內建支援：thumbs_up, thumbs_down, open_palm, closed_fist, victory(✌️), pointing_up, ILoveYou(🤟)
5. **每個手勢的 TTS 回應要說什麼？**

**請填這個表交回來**：

| 手勢 | Go2 動作 (api_id) | TTS 語音 | 冷卻時間 |

| thumbs_up | ?（Content? Dance1? WiggleHips?） | ?（「謝謝！」?「你真棒！」?） | ? |
| wave | ?（Hello? 其他?） | ?（「你好！」?） | ? |
| ok | ?（Content? 其他?） | ? | ? |
| point | ?（轉頭? 不做?） | ? | ? |
| victory(✌️) | ? | ? | ? |
| thumbs_down | ? | ? | ? |
| ＿＿（你想加的） | ? | ? | ? |

**已知限制**：
- 有效距離約 **2 公尺**，距離過遠不精準
- **僅支援單人**：多人同時出現會混亂

### 🧍 姿勢辨識（陳若恩 — 如果語音忙不過來，由其他人接）

**目前有的（只有 1 個映射）**：

| 姿勢 | Go2 動作 | TTS | 冷卻 |
|------|---------|-----|:----:|
| fallen（跌倒） | — | 「偵測到跌倒！請注意安全」 | 10s |
| standing / sitting / crouching | 無 | 無 | — |

**你要思考的**：

1. **站立 (standing) → 要不要做什麼？** 例如看到人站著就打招呼？
2. **坐下 (sitting) → 要不要做什麼？** 例如 Go2 也跟著坐下？
3. **蹲下 (crouching) → 要不要做什麼？** 例如 Go2 降低身高？
4. **跌倒偵測要不要保留？** 幻覺問題嚴重（無人時也會誤判），Demo 時可關閉
5. **姿勢變化的組合場景？** 例如：站→蹲→站 = 做了蹲下動作 → Go2 模仿？

**請填這個表交回來**：

| 姿勢 | Go2 動作 (api_id) | TTS 語音 | 冷卻時間 |
|------|-------------------|---------|:--------:|
| standing | ?（BalanceStand? StandUp?） | ? | ? |
| sitting | ?（Sit? 不做?） | ? | ? |
| crouching | ?（StandDown 降低身高?） | ? | ? |
| fallen | StopMove(1003) + ?（RecoveryStand?） | 「偵測到跌倒！請注意安全」 | 10s |
| ＿＿（你想加的組合） | ? | ? | ? |

**已知限制**：
- **僅支援單人追蹤**
- **幻覺問題**：無人時偶爾誤判有人跌倒（鎖定衣架等物體）
- 鏡頭高度 **~1.5m** 才看得到全身
- 側面坐姿判定不準，Demo 時建議正面面向攝影機
- `enable_fallen` 參數可關閉跌倒偵測（避免 Demo 誤報）

### 🗣 語音功能（陳若恩 — 主責）

**目前有的**：

| 語音意圖 | Go2 動作 | 說明 |
|---------|---------|------|
| greet（你好） | Hello(1016) | 打招呼 |
| stop（停） | StopMove(1003) | 停止 |
| sit（坐下） | Sit(1009) | 坐下 |
| stand（站起來） | StandUp(1004) | 站起來 |
| chat（其他） | 無動作 | LLM 回覆 |

**你要做的（連接 GPU 雲端）**：

1. **LLM 回答要更聰明**：目前 max_tokens=120、限制 25 字，回答像機器人。要放寬長度、加入自我介紹、個性化回答
2. **多輪對話**：目前每次都是獨立回答，沒有記住上一句。要加 conversation history
3. **固定腳本 (Plan B)**：GPU 斷線時的備案。設計 10-15 組固定問答：

**Plan B 固定腳本（請設計）**：

| 使用者說 | Go2 回答 | Go2 動作 |
|---------|---------|---------|
| 你好 | ? | Hello(1016) |
| 你叫什麼名字 | ?（「我叫 PawAI，是一隻會說話的機器狗！」） | ? |
| 你有什麼功能 | ? | ? |
| 坐下 | ? | Sit(1009) |
| 站起來 | ? | StandUp(1004) |
| 跳舞 | ? | Dance1(1022)? |
| 你幾歲 | ? | ? |
| 再見 | ? | Hello(1016)? |
| ＿＿ | ? | ? |

**非常取決於 GPU 狀態**：
- 雲端正常 → AI 自由對話（~2s 回應）
- 雲端斷線 → 自動切 Plan B 固定台詞（~0.x 秒）
- **本地 ASR 不可用**（噪音干擾嚴重，長句辨識失敗）
- **本地 LLM 不可用**（Qwen 0.8B 智商極低）

**遠端開發**：語音功能全走雲端，你只需要 SSH 連上 RTX 8000 server 調整 LLM prompt 和固定台詞。不需要 Go2。

## 互動場景設計（組合場景）

目前每個功能各自獨立，缺乏**複合場景**。以下是可以設計的組合：

### Demo 完整流程

```
1. 人走近（距離 < 2m）
   → 人臉辨識到 → Hello(1016) + TTS「你好，[名字]！」
2. 人比 thumbs_up
   → 手勢辨識 → Content(1020) + TTS「謝謝！」
3. 人說「坐下」
   → 語音辨識 → Sit(1009) + TTS「好的！」
4. 人展示杯子
   → 物體辨識 → TTS「你要喝水嗎？」
5. 人比 stop
   → 手勢辨識 → StopMove(1003)
6. 人說「再見」
   → 語音辨識 → Hello(1016) + TTS「再見！」
```

### 猜拳遊戲（Roy 建議的互動）

```
人比出拳頭/剪刀/布 → Go2 隨機出一個 + TTS「我出布！你輸了！」
```

需要新增 rock/scissors/paper 手勢辨識。
