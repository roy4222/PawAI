**根本原因分析：**

   - 包包：布料/皮革（反射不規則）

3. **單一校正參數局限**
   - 當前：所有物件用同一組 SCALE 參數
   - 應該：不同物件使用不同校正參數

### 方案 A：主動探索模式 🔴 必做

**核心思想：** 找不到物品 → 移動 → 再找 → 重複（最多 N 次）

**實作架構：**

```python
# 新增檔案：ros-mcp-server/active_search.py

class ActiveSearchManager:
    """主動探索管理器"""

    async def search_with_exploration(
        self,
        target: str,
        max_attempts: int = 5,
        strategy: str = "rotate_scan"
    ) -> dict:
        """
        策略選項：
        - rotate_scan: 原地旋轉 360 度
        - patrol_points: 移動到預設巡邏點
        """
        for attempt in range(max_attempts):
            result = await find_object(target)
            if result["found"]:
                return result

            if strategy == "rotate_scan":
                await self.rotate(angle=60)  # 旋轉 60 度

            await asyncio.sleep(1.0)  # 等待穩定

        return {"success": False, "message": "搜尋失敗"}
```

**新增 MCP Tool：**

```python
@mcp.tool()
async def find_object_with_exploration(
    target: str,
    max_attempts: int = 5,
    exploration_strategy: str = "rotate_scan"
) -> dict:
    """主動尋物（含探索）"""
    manager = ActiveSearchManager()
    return await manager.search_with_exploration(
        target, max_attempts, exploration_strategy
    )
```

### 方案 B：物件校正表 🟡 建議做

**核心思想：** 不同物件使用不同的校正參數

**校正表設計：**

```python
# 修改檔案：GPU Server 上的 perception_server_v2.py

CALIBRATION_TABLE = {
    'chair': {
        'thresholds': [0.80, 3.00],
        'scales': [0.66, 0.23, 0.20],
        'blend_width': 0.20
    },
    'bottle': {
        'thresholds': [0.60, 2.50],
        'scales': [0.55, 0.20, 0.18],  # 待測量
        'blend_width': 0.15
    },
    'cup': {
        'thresholds': [0.50, 2.00],
        'scales': [0.50, 0.18, 0.15],  # 待測量
        'blend_width': 0.15
    },
    'bag': {
        'thresholds': [0.70, 2.80],
        'scales': [0.45, 0.18, 0.15],  # 待測量
        'blend_width': 0.20
    },
    'default': {  # 未知物件使用泛化參數
        'thresholds': [0.70, 2.50],
        'scales': [0.55, 0.20, 0.18],
        'blend_width': 0.20
    }
}

def calibrate_depth(depth_raw: float, object_label: str) -> float:
    """根據物件類型進行深度校正"""
    params = CALIBRATION_TABLE.get(object_label, CALIBRATION_TABLE['default'])
    thresholds = params['thresholds']
    scales = params['scales']
    blend_width = params['blend_width']

    # 分段線性校正（平滑過渡）
    if depth_raw < thresholds[0] - blend_width:
        scale = scales[0]
    elif depth_raw < thresholds[0] + blend_width:
        blend = (depth_raw - (thresholds[0] - blend_width)) / (2 * blend_width)
        scale = scales[0] * (1 - blend) + scales[1] * blend
    elif depth_raw < thresholds[1] - blend_width:
        scale = scales[1]
    elif depth_raw < thresholds[1] + blend_width:
        blend = (depth_raw - (thresholds[1] - blend_width)) / (2 * blend_width)
        scale = scales[1] * (1 - blend) + scales[2] * blend
    else:
        scale = scales[2]

    return depth_raw * scale
```

## 🔧 技術細節

### 旋轉搜尋邏輯

```python
async def rotate_scan(self, target: str, angle_step: int = 60):
    """原地旋轉 360 度搜尋"""
    
    total_rotation = 0
    max_rotation = 360
    
    while total_rotation < max_rotation:
        # 1. 當前角度拍照
        result = await find_object(target)
        if result["found"]:
            return result
        
        # 2. 旋轉到下一個角度
        await self.publish_cmd_vel(
            linear_x=0.0,
            angular_z=0.3,  # 正值 = 逆時針
            duration=angle_step / 60.0  # 60°/s → 持續時間
        )
        
        total_rotation += angle_step
        await asyncio.sleep(1.0)  # 等待穩定
    
    return {"success": False, "message": "360° 搜尋完成，未找到"}
```
