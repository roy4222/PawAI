## 🔴 關鍵釐清：`connect_to_robot` vs `start_mcp.sh`

**結論：完全不同！兩者都需要。**

| 項目         | `start_mcp.sh`                | `connect_to_robot()` |
| ------------ | ----------------------------- | -------------------- |
| **功能**     | 啟動整個 ROS2 生態系統        | 只設定連線 IP/Port   |
| **執行時機** | VM 上先跑                     | LLM 呼叫 MCP Tool    |
| **啟動內容** | rosbridge + driver + services | 無（只 ping 測試）   |

**正確流程：**
```
1️⃣ VM 上先跑 start_mcp.sh → 啟動 rosbridge + driver + services
2️⃣ LLM 呼叫 connect_to_robot("192.168.1.200", 9090) → MCP 知道連哪裡
3️⃣ LLM 呼叫其他 MCP Tools → 實際控制機器狗
```

## ⚡ 延遲優化策略

### 延遲來源分析

```
總延遲 ≈ 3-5 秒
├── 🐢 LLM 推論時間    1-3 秒   ← 瓶頸！
├── 🐇 MCP Tool 執行   50-100 ms
├── 🐇 rosbridge      10-30 ms
└── 🐇 ROS2 執行      100-500 ms
```

### LangGraph 能降低延遲嗎？

**結論：不能。** LangGraph 解決的是**智能編排**，不是**延遲**。

### 真正能降低延遲的方法

| 方法                                  | 效果  | 說明                      |
| ------------------------------------- | ----- | ------------------------- |
| **合併多 Tool 為複合 Tool**           | ⭐⭐⭐⭐⭐ | 一次呼叫完成多步驟        |
| **用 Perception API 取代 LLM Vision** | ⭐⭐⭐⭐⭐ | 已在做！3-7s → 1.5-2.5s   |
| **精簡 System Prompt**                | ⭐⭐⭐⭐  | 減少 Token = 減少推論時間 |
| **自架網站取代 Kilo Code**            | ⭐⭐⭐   | 去掉 UI 開銷              |
| **本地 LLM**                          | ⭐⭐⭐   | 省掉 API 網路延遲（寒假） |

### 改造 2：複合 Tool（`scan_and_move`）

```python
@mcp.tool()
async def scan_and_move() -> dict:
    """拍照→分析→自動移動，一次完成"""
    snapshot = await call_service("/capture_snapshot", ...)
    perception = await http_post(PERCEPTION_URL, image=snapshot)
    
    # 邏輯寫死在 Python，不讓 LLM 思考
    if perception["front_obstacle_m"] > 1.0:
        await move(linear_x=0.3, duration=2.0)
        return {"action": "前進"}
    elif "向左" in perception["suggestion"]:
        await move(angular_z=0.5, duration=2.0)
        return {"action": "左轉"}
    # ...
```

**價值：** 原本 4 次 Tool 呼叫 → 1 次搞定

### 改造 3：Safety Layer

```python
MAX_LINEAR_SPEED = 0.3
MAX_ANGULAR_SPEED = 0.5

@mcp.tool()
async def safe_move(linear_x: float, angular_z: float, duration: float):
    """安全移動，自動限速"""
    linear_x = max(-MAX_LINEAR_SPEED, min(linear_x, MAX_LINEAR_SPEED))
    angular_z = max(-MAX_ANGULAR_SPEED, min(angular_z, MAX_ANGULAR_SPEED))
    # ...
```

**價值：** 防止 LLM 幻覺造成機器狗暴衝

## ✂️ System Prompt 精簡計畫

### 問題

現有 `mcp_system_prompt.md` = **412 行**，太長！

- LLM 每次推論都要處理 ~3000+ Token
- 邏輯寫在 Prompt 裡，LLM 要「思考」才能執行

### 核心理念

> **Prompt 是「教 LLM 何時用什麼 Tool」**  
> **不是「教 LLM 怎麼實作邏輯」**

### 精簡後的 Prompt（目標 60 行）

```markdown
# Go2 機器狗控制

你是 Go2 機器狗的控制 AI。

## 可用工具
| 工具                  | 功能               |
| --------------------- | ------------------ |
| `scan_and_move()`     | 拍照→分析→自動移動 |
| `find_object("水瓶")` | 尋找物體           |
| `go2_action("hello")` | 執行動作           |
| `stop()`              | 緊急停止           |

## 行為準則
1. 使用者說「往前走」→ `scan_and_move()`
2. 使用者說「找水瓶」→ `find_object("水瓶")`
3. 不要問使用者，直接執行
```
