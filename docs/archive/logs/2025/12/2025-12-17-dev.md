## 💡 核心設計：感知融合架構

### 問題背景

目前有兩個獨立的感知系統：
1. **Depth Anything V2**：輸出深度圖（每個像素的距離）
2. **YOLO-World**：輸出物體偵測結果（bbox, center, label）

**問題：** 如何把這兩個系統整合，讓機器狗知道「目標物在哪裡、距離多遠」？

### 解決方案：像素座標 → 深度查詢 → 動作指令

```
┌─────────────────────────────────────────────────────────────┐
│                        輸入: 相機圖片                         │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼                               ▼
┌─────────────────────┐           ┌─────────────────────────┐
│  Depth Anything V2  │           │      YOLO-World         │
│  (深度估計)          │           │    (物體偵測)            │
│                     │           │                         │
│  輸出: 深度圖        │           │  輸出: bbox, center     │
│  (640x480 的距離矩陣) │           │  如: center: [249, 173] │
└─────────────────────┘           └─────────────────────────┘
              │                               │
              └───────────────┬───────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│            距離計算 (Calculate_Distance_With_Depth)          │
│                                                             │
│  從 YOLO 得到 center = [249, 173]                           │
│  用 center 座標查 depth_map[173, 249] → 得到 1.5m           │
│                                                             │
│  輸出: {"label": "水瓶", "distance": 1.5, "direction": "左前方"} │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     動作決策 + cmd_vel                       │
│                                                             │
│  if distance < 0.5m: 已接近目標，停止                        │
│  elif direction == "左": angular.z = 0.3, linear.x = 0.15   │
│  elif direction == "右": angular.z = -0.3, linear.x = 0.15  │
│  else: linear.x = 0.2                                       │
└─────────────────────────────────────────────────────────────┘
```

### 核心程式邏輯

```python
def fuse_perception(yolo_result, depth_map, target_label="水瓶"):
    """
    融合 YOLO 偵測結果與深度圖
    
    Args:
        yolo_result: YOLO-World 回傳的 JSON
        depth_map: Depth Anything 回傳的深度矩陣 (H x W)
        target_label: 想找的物體
    
    Returns:
        目標物的距離與方位
    """
    image_width = 640
    SCALE_FACTOR = 0.60  # Go2 廣角鏡頭校正係數
    
    # 1. 從 YOLO 結果找目標物
    target = None
    for obj in yolo_result["results"]:
        if obj["label"] == target_label and obj["score"] > 0.2:
            target = obj
            break
    
    if target is None:
        return {"found": False, "suggestion": "旋轉掃描環境"}
    
    # 2. 取得物體中心的像素座標
    cx, cy = target["center"]  # 例如 [249, 173]
    
    # 3. 從深度圖查詢該點的距離
    # 注意：depth_map 是 [高度, 寬度] 格式
    distance = depth_map[cy, cx] * SCALE_FACTOR
    
    # 4. 判斷物體在畫面的哪一側（決定轉向方向）
    if cx < image_width / 3:
        direction = "左側"
        angular_z = 0.3   # 左轉
    elif cx > image_width * 2 / 3:
        direction = "右側"
        angular_z = -0.3  # 右轉
    else:
        direction = "正前方"
        angular_z = 0.0   # 直走
    
    return {
        "found": True,
        "label": target_label,
        "distance_m": round(distance, 2),
        "direction": direction,
        "pixel_center": [cx, cy],
        "cmd_vel": {
            "linear": {"x": 0.15 if distance > 0.5 else 0.0, "y": 0.0, "z": 0.0},
            "angular": {"x": 0.0, "y": 0.0, "z": angular_z}
        }
    }
```

### API 設計建議

擴展現有 Perception Server（Port 8050）：

```python
@app.post("/find_object")
async def find_object(image: UploadFile, target: str = "水瓶"):
    # 1. 同時運行 DA3 + YOLO-World
    depth_map = run_depth_estimation(image)
    yolo_result = run_yolo_world(image, classes=[target])
    
    # 2. 融合結果
    result = fuse_perception(yolo_result, depth_map, target)
    
    # 3. 回傳給機器狗
    return result
```


### 會議重點

1. **影像與硬體限制**
   - 機器狗鏡頭 1080p/720p，網路傳輸會壓縮
   - 巨量模型（200B-300B）無法運行，需用輕量模型

2. **架構演進確認**
   - 純 VLM（圖片 → LLM）成功率僅 5-6 成，延遲高
   - 新架構：DA3 + YOLO → JSON → LLM，快 3 倍

3. **專案目標擴展**
   > 「尋物功能似不太能發揮亮點，我們再一起想想如何讓 Go2 用 MCP 發揮更多用途」
   
   - **新定位**：從「尋物工具」→「**智能、互動式 AI 機器狗**」
   - 讓機器狗「聽得懂人話」，調用動作，實現多元互動
   - 尋物只是功能之一，不是全部

4. **Demo 策略**
   - 1/7 以**錄影為主**（網路環境考量）
   - 簡報著重「架構演進」，而非僅展示最終功能
   - Isaac Sim 作為進階功能與高風險場景展示

5. **長期方向**
   - 改用輕量本地模型（Mistral 14B / Qwen 8B-32B Light）
   - 降低 API 延遲與成本
