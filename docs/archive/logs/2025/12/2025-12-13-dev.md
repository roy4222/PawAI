## ⚠️ 純視覺方案的關鍵問題

> **重要發現（測試過程中觀察到）：**
> 
> **問題：LLM 無法認知機器狗自身大小**
> - 辨識障礙物 ✅ 成功
> - 避障決策 ✅ 會轉向
> - **判斷是否可通過 ❌ 失敗**
> 
> **症狀：** 機器狗只轉一點點就說「安全了」，但實際上機體仍會撞到障礙物
> 
> **根因：** 純視覺無法提供精確的空間感知，LLM 不知道「自己有多大」
> 
> **結論：** 純視覺方案對於「粗略避障」可行，但精確導航仍需 **SLAM + Nav2**

> 🎉 **Amazon Nova 2 Lite 測試結果：**
> - **優點：** 正確識別前方障礙物，建議左轉
> - **小問題：** 物體判斷錯誤（灰色劍判斷成白色紙箱）
> - **成本：** 🆓 **免費！**
> - **結論：** ✅ **目前最佳方案！**

> ✅ **Claude Sonnet 4.5 測試結果：**
> - **優點：** 正確識別前方障礙物，建議右轉 30-45 度
> - **小問題：** 顏色判斷錯誤（灰色劍判斷成白色圓柱）
> - **成本：** $0.42/次（太貴！10 次就 $4.2）
> - **結論：** 🟡 功能可用，但成本過高

> ⚠️ **Claude 4.5 Haiku 測試結果：**
> - **致命問題：** 畫面正中央有一把灰色的劍/支架，LLM 卻判定「完全清空，無障礙物」
> - **成本：** $0.18/次
> - **結論：** ❌ 不可用

**結論：** 🎉 **Amazon Nova 2 Lite 是目前最佳方案（免費 + 正確識別）！**

```diff
# 避障策略（新版 - 自主模式）
+ 如果前方有障礙物 → **直接執行繞行**
+ 繞行後告知：「我發現障礙物，已右轉繞開了！」
```

## 💡 執行小技巧

### Tip 1: Phase 1 強制 JSON 輸出
```
分析圖片。回傳 JSON 格式：
{"obstacle": bool, "description": "簡短說明", "action": "建議動作"}
不要輸出其他文字。
```
> 優點：生成 JSON 比自然語言快，Token 數少

### Tip 2: Phase 2 檢查 Odom 邏輯
- 如果是 `sleep(time)` → 不準（加減速需要時間）
- 如果是 `while (current_dist < target)` → 檢查 cmd_vel 頻率

### Tip 3: Phase 3 定義轉向幅度
在 Prompt 定義默認值：
- 「向右轉」→ 默認 30 度
- 避免機器狗原地轉圈

## 🆕 今日新增功能

### `/move_for_duration` 服務

**新增檔案：**
- `go2_interfaces/srv/MoveForDuration.srv` - 服務定義
- `go2_robot_sdk/move_service.py` - Python 服務節點

**功能：**
```python
call_service('/move_for_duration', 'go2_interfaces/srv/MoveForDuration', 
             {"linear_x": 0.2, "angular_z": 0.0, "duration": 3.0})
```

**安全機制：**
- 速度上限：linear ±0.3 m/s, angular ±0.5 rad/s
- 時間上限：10 秒
- 提供 `/stop_movement` 緊急停止服務
- 使用 ROS Clock 計時，避免時間漂移

### rosbridge 修正

**問題：** rosbridge 未 source workspace，無法識別自訂服務類型

**修正：** 在 `start_mcp.sh` 中加入 source：
```bash
source $SCRIPT_DIR/install/setup.zsh  # 新增這行
ros2 launch rosbridge_server rosbridge_websocket_launch.xml
```
