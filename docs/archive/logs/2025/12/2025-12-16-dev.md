## 💡 架構定位討論

### 核心問題：我們的架構是 VLA 嗎？

**結論：不是 VLA，而是「輕量級雙系統架構」**

| 特徵 | 真正的 VLA | 我們的架構 |
|-----|-----------|-----------|
| 動作輸出 | LLM 直接輸出 action tokens | LLM 輸出文字 → 轉譯為服務呼叫 |
| 訓練方式 | 微調 VLM 生成動作 | 零樣本推論 |
| 特點 | 端到端 | 模組化 |

### 我們的架構分類

```
「LLM for Robotics」(3.4) + 「零樣本導航」(3.9)
= 輕量級雙系統架構

System 2 (慢系統): LLM 語意理解
System 1 (快系統): 確定性控制腳本
```

---

## ⚡ 低延遲感知策略

### 原架構 vs 新架構

| 方案 | 流程 | 延遲 |
|-----|------|------|
| 原本 | Camera → LLM Vision → 決策 | 3-7 秒 |
| **現在** | Camera → DA3 → JSON → LLM Text → 決策 | **1.5-2.5 秒** |

### 設計理念

> **不給 LLM 吃「生肉」（圖片），給 LLM 吃「熟食」（JSON 摘要）**

優勢：
- 🚀 快 3 倍
- 💰 Token 成本降低 90%
- 🔧 可調試（JSON 中間結果）

## 🧠 記憶系統設計

### 核心問題：連續動作 + 目標記憶

```
問題：
看到水瓶 → 避障轉左 → 看不到水瓶了 → 忘記目標？ ❌

解決：
引入「短期記憶」追蹤任務狀態
```

### 記憶層次

| 層次 | 內容 | 生命週期 | 1/7 需要？ |
|-----|------|---------|-----------|
| 短期記憶 | 目標、位置、動作歷史 | 任務期間 | ✅ 必須 |
| 中期記憶 | 今天找過的物品 | 程式運行期間 | 🟡 可選 |
| 長期記憶 | 物品通常位置 | 永久 (SQLite) | ❌ 寒假 |

### 最小實作

```python
class ShortTermMemory:
    goal: str           # "找水瓶"
    target: dict        # {"direction": "right", "distance": 2.0}
    phase: str          # SEARCHING → APPROACHING → AVOIDING → FOUND
    actions: list       # 最近動作
```

## 📝 關鍵決策記錄

### 決策 1：架構定位
- **選擇**：輕量級雙系統（非 VLA）
- **理由**：可控、可解釋、低延遲

### 決策 2：感知策略
- **選擇**：Perception API (DA3 + YOLO) → JSON → LLM
- **理由**：比 LLM Vision 快 3 倍

### 決策 3：記憶系統
- **選擇**：短期記憶 + 任務狀態機
- **理由**：解決連續動作和目標追蹤問題

### 決策 4：1/7 Demo 範圍
- **選擇**：跳過 SLAM/Nav2，專注視覺尋物
- **理由**：簡化複雜度，確保 Demo 成功

### 決策 5：長期架構
- **選擇**：LangGraph 多代理人 + 雙系統
- **時程**：寒假開始引入
