### [T1] 解決 DDS 通訊與數據丟失問題
- **狀態**：**重大突破 (Major Breakthrough)**

## 🐛 遇到的問題 (Issues & Bugs)

### 🔴 問題 1：SCTP 握手超時 (SCTP Handshake Timeout)
- **相關任務**：[T1]
- **嚴重程度**：**Critical** (阻斷所有傳感器數據)
- **現象描述**：
  Driver Log 顯示：
  ```text
  [go2_driver_node] error_received([Errno 101] Network is unreachable)
  [go2_driver_node] ERROR - [診斷] ❌ SCTP 握手超時（>30.0秒）
  ```
- **原因**：VM 橋接網卡 `enp0s1` 同時存在兩個 IP (Static + DHCP)，導致 WebRTC 封包發送源頭不確定。
- **最終解決方案**：
  在啟動前執行強力清洗：
  ```bash
  sudo systemctl stop NetworkManager
  sudo killall dhclient
  sudo ip addr flush dev enp0s1
  sudo ip addr add 192.168.12.222/24 dev enp0s1
  ```

### 🟠 問題 2：ROS2 平行時空 (Discovery but no Data)
- **相關任務**：[T1]
- **嚴重程度**：Major
- **現象描述**：
  T1 正常運行且有數據，但 T2 `ros2 topic list` 有東西，`ros2 topic hz` 卻什麼都收不到。
- **原因**：網路介面重置後，ROS2 Daemon 仍緩存舊的網路狀態，導致節點綁定錯誤。
- **最終解決方案**：
  1. 確保 `CYCLONEDDS_URI` 指向 `local_only_v2.xml` (Loopback)。
  2. 重置網路後執行 `ros2 daemon stop`。

## 📚 學習與筆記 (Notes)

- **除錯技巧**：當 ROS2 看得到 Topic 但收不到 Data 時，先檢查 `ros2 run demo_nodes_cpp talker/listener`，並確認 XML 設定檔是否強制了 Loopback。
