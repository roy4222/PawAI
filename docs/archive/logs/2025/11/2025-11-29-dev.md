### 📊 技術收穫

#### 1. ROS2 Windows Binary Release 版本管理知識

**學習重點：**
- ROS2 每個發行版（如 Humble）會持續推出 **Patch Release**
- Patch Release 只修 bug，不改 API，版本間**完全兼容**
- Windows Binary 在某些版本可能有打包問題

**版本選擇策略：**
```
最新版本（激進）  vs  穩定版本（保守）
     ↓                      ↓
Patch 13 (375 MB?)    Patch 7 (1.8 GB ✅)
可能有問題            社群驗證充分
```

**決策依據：**
- **Mac VM 版本**：2025-05（相當新）
- **Windows 版本選擇**：2023-06-14（穩定）
- **兼容性**：只要都是 Humble，patch 版本間完全兼容 ✅

---

#### 2. Windows ROS2 安裝與 Linux 的差異

| 項目 | Linux (Mac VM) | Windows |
|------|---------------|---------|
| **安裝方式** | `apt install ros-humble-desktop` | Binary ZIP 解壓縮 |
| **Python 版本** | 3.10 | **3.8.3**（強制） |
| **套件管理** | apt | Chocolatey |
| **Python 擴展副檔名** | `.so` | **`.pyd`** |
| **環境變數** | 自動配置 | 需手動設定 `.bat` 腳本 |
| **Qt 安裝** | 通過 apt 自動安裝 | **需手動下載 3 GB 離線安裝器** |

**關鍵差異：**
- Windows 需要更多**手動配置**
- 依賴版本要求**更嚴格**（Python 必須 3.8.3）
- Qt5 是**最容易遺漏的關鍵依賴**

---

#### 3. 套件管理工具選擇：uv vs pip

**為什麼選擇 uv？**

```
pip（傳統）               uv（現代）
    ↓                        ↓
速度慢                   快 10-100 倍
依賴解析慢               智能依賴解析
單線程下載               並行下載
```

**實際使用差異：**
```powershell
# pip（舊方式）
python -m pip install catkin_pkg cryptography empy lark lxml netifaces numpy pyparsing pyyaml
# 預計時間：2-3 分鐘

# uv（新方式）
uv pip install --system catkin_pkg cryptography empy lark lxml netifaces numpy pyparsing pyyaml
# 預計時間：10-20 秒 ⚡
```

**注意事項：**
- `--system` 參數：安裝到系統 Python（不是虛擬環境）
- 與 pip 完全兼容，可混用

#### 4. 文件驗證的重要性

**學到的教訓：**

**錯誤做法：**
```
看到檔案存在 → 直接解壓縮 → 安裝失敗 → 浪費時間除錯
```

**正確做法：**

```powershell
# 2. 檢查解壓後大小
$installSize = (Get-ChildItem C:\dev\ros2_humble -Recurse | Measure-Object -Property Length -Sum).Sum / 1GB
# 預期：> 2.0 GB ✅

# 3. 檢查關鍵檔案
```

## 💭 開發反思

### 今日學到的最重要的事

#### 1. **驗證比安裝更重要**

**錯誤做法（今天踩的坑）：**
```
下載檔案 → 直接解壓縮 → 發現問題 → 浪費時間除錯
```

**正確做法（教訓）：**
```
下載檔案 → 檢查大小 → 解壓縮 → 驗證關鍵檔案 → 測試基本功能
```

**啟示：**
- 每個步驟都要有**驗收標準**
- 問題越早發現，修復成本越低
- 自動化驗證腳本可以節省大量時間

---

#### 2. **官方文件不一定完整**

**發現的問題：**
- ROS2 官方文件沒有強調 **Qt5 是必要依賴**
- 沒有提供 **檔案大小檢查** 的建議
- 沒有說明 **最新版本可能有問題**

**解決方案：**
- 參考**社群經驗**（GitHub Issues, ROS Answers）
- 建立**自己的驗證清單**
- 記錄**踩過的坑**（寫詳細的開發日誌）

---

#### 3. **版本選擇的策略**

**激進 vs 保守：**

```
最新版本（激進）         穩定版本（保守）
       ↓                        ↓
  可能有新 bug             bug 已知且有解法
  社群經驗少               社群經驗多
  文件可能不完整           文件完整

  適合：實驗、學習         適合：生產、趕進度
```

**今日決策：**
- 選擇 **Patch Release 7 (2023-06-14)** ✅
- 原因：需要**趕進度**完成 Phase 1 測試
- 代價：錯過最新的 bug 修復（可接受）

---

#### 4. **工具選擇的重要性**

**uv vs pip 的實際差異：**

```
任務：安裝 8 個 Python 套件

pip 方式：
  下載速度：慢（單線程）
  依賴解析：慢（暴力嘗試）
  總時間：2-3 分鐘

uv 方式：
  下載速度：快（並行下載）
  依賴解析：快（智能算法）
  總時間：10-20 秒

時間節省：90% ⚡
```

**啟示：**
- 選對工具可以節省 **10 倍時間**
- 新工具不一定不穩定（uv 已經很成熟）
- 值得花時間學習**現代化工具**

### 明日注意事項

#### 1. **下載 ROS2 Binary 時**

**檢查清單：**
- ☐ 確認檔案大小 > 1.5 GB
- ☐ 確認下載完整（100%）
- ☐ 確認檔案名稱正確（包含 `windows-release-amd64`）
- ☐ 確認下載來源（GitHub Releases 官方頁面）

**驗證腳本（下載後立即執行）：**
```powershell
$file = Get-Item "$env:USERPROFILE\Downloads\ros2-humble-*.zip"
$sizeGB = [math]::Round($file.Length / 1GB, 2)
if ($sizeGB -lt 1.5) {
    Write-Host "❌ 檔案異常！重新下載" -ForegroundColor Red
} else {
    Write-Host "✅ 檔案正常（$sizeGB GB）" -ForegroundColor Green
}
```

---

#### 2. **解壓縮 ROS2 時**

**注意事項：**
- 確保目標路徑 **沒有中文字元**
- 確保目標路徑 **沒有空格**
- 解壓縮時間較長（2-5 分鐘），**不要中斷**
- 解壓後立即**驗證關鍵檔案**

**驗證腳本（解壓後立即執行）：**
```powershell
$checks = @{
    "local_setup.bat" = Test-Path "C:\dev\ros2_humble\local_setup.bat"
    "rclpy" = Test-Path "C:\dev\ros2_humble\Lib\site-packages\rclpy"
    "_rclpy_pybind11.pyd" = (Get-ChildItem "C:\dev\ros2_humble\Lib\site-packages" -Filter "*_rclpy_pybind11*" -Recurse).Count -gt 0
}

$allPassed = $true
foreach ($key in $checks.Keys) {
    if (-not $checks[$key]) {
        Write-Host "❌ $key 檢查失敗" -ForegroundColor Red
        $allPassed = $false
    }
}

if ($allPassed) {
    Write-Host "✅ 所有檢查通過" -ForegroundColor Green
} else {
    Write-Host "❌ 安裝不完整，請重新解壓縮" -ForegroundColor Red
}
```

**故障排查順序：**
```
1. ping 192.168.1.200（網路層）
   ↓ 通過
2. 檢查環境變數（應用層）
   ↓ 通過
3. ros2 topic list（ROS2 層）
   ↓ 通過
4. ros2 topic hz /scan（數據層）
   ↓ 通過
5. rviz2（視覺化層）
```

**每一層測試通過再進入下一層！**
