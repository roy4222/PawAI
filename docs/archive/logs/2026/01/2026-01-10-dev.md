## 📅 今日重點：Go2 Pro 有線模式測試與發現

今天嘗試建立 **Mac VM 與 Go2 的 CycloneDDS 有線連接**，雖然網路層連通成功，但發現 **Go2 Pro 的架構限制**導致無法透過有線取得 ROS2 Topics。

## 🔌 一、網路架構配置

### 最終連線架構

```
Windows PC (SSH Client)
    ↓ SSH (192.168.0.149)
Mac VM (Ubuntu 22.04, UTM)
    ├── enp0s1: 192.168.0.149/24 (Wi-Fi 橋接，供 SSH 使用)
    └── enp0s2: 192.168.123.100/24 (乙太網橋接 → en5 → Go2)
           ↓ 有線連接 (<1ms 延遲)
Go2 MCU (192.168.123.161)
```

### 關鍵設備 IP

| 設備 | IP | 用途 |
|------|-----|------|
| Mac VM (Wi-Fi) | 192.168.0.149 | SSH 遠端連線 |
| Mac VM (乙太網) | 192.168.123.100 | Go2 控制通道 |
| Mac 主機 (乙太網) | 192.168.123.99 | 橋接中繼 |
| Go2 MCU | 192.168.123.161 | 機器狗主控 |

---

## 🔧 二、設定步驟記錄

### 1. Mac 主機設定

```bash
# 設定 en5 乙太網介面 IP
sudo ifconfig en5 192.168.123.99 netmask 255.255.255.0

# 驗證 Go2 連通
ping -c 4 192.168.123.161
# 結果：0.5-0.8ms，連通成功 ✅
```

### 2. UTM 虛擬機設定

- **Network 1 (enp0s1)**: Bridged → en0 (Wi-Fi)，用於 SSH
- **Network 2 (enp0s2)**: Bridged → en5 (乙太網)，用於 Go2

### 3. VM 內部 IP 配置

```bash
# 修正 enp0s2 的 IP（原本錯誤設為 192.168.1.200）
sudo ip addr del 192.168.1.200/24 dev enp0s2
sudo ip addr add 192.168.123.100/24 dev enp0s2

# 驗證 Go2 連通
ping -c 4 192.168.123.161
# 結果：0.5-1.3ms，連通成功 ✅
```

## ⚠️ 三、遇到的問題與解法

### 問題 1：VM ping 不到 Go2

**症狀：**
```
From 192.168.123.100 icmp_seq=1 Destination Host Unreachable
```

**原因：**
1. Mac 主機的 en5 沒有設定 IP（只有 link-local 169.254.x.x）
2. VM 的 enp0s2 設定了錯誤的 IP (192.168.1.200)

**解法：**
1. Mac 主機：`sudo ifconfig en5 192.168.123.99 netmask 255.255.255.0`
2. VM：`sudo ip addr add 192.168.123.100/24 dev enp0s2`

### 問題 2：UTM 橋接設定

**症狀：** 即使 Mac 主機能 ping Go2，VM 仍無法連通

**解法：** 確認 UTM Network 2 橋接到 `en5`（連接 Go2 的乙太網介面）

## 🔴 七、重大發現：Go2 Pro 有線模式限制

### Go2 內部網路架構

```
┌─────────────────────────────────────────────────────┐
│                    Go2 Pro 內部                      │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │ Wi-Fi 介面 (192.168.12.1)                   │   │
│  │  └─ WebRTC 服務                              │   │
│  │      ├─ 影像串流 (/camera/image_raw)        │   │
│  │      ├─ LiDAR 點雲 (/point_cloud2)          │   │
│  │      └─ IMU 資料 (/imu)                     │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │ 有線介面 (192.168.123.161)                  │   │
│  │  └─ MCU 控制                                 │   │
│  │      ├─ 運動控制指令                         │   │
│  │      └─ 韌體更新                             │   │
│  │  ⚠️ **不發送 ROS2 Topics**                   │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  ❌ 兩個網段內部不互通                              │
└─────────────────────────────────────────────────────┘
```

### 測試驗證

```bash
# 嘗試從有線網段連接 WebRTC
ping 192.168.12.1
# 結果：100% packet loss - 無法連通

# 確認：有線網段只能到達 MCU
ping 192.168.123.161
# 結果：0.5ms - 連通成功
```

### 結論

| 版本 | 有線模式支援 | 說明 |
|------|-------------|------|
| **Go2 Pro** | ❌ 不完整 | 有線只能到 MCU，無 ROS2 Topics |
| **Go2 EDU** | ✅ 完整 | 內建 Jetson + ROS2，支援 CycloneDDS |

---

## 📋 八、可行的替代方案

### 方案 1：繼續使用 Wi-Fi WebRTC（推薦）

- 使用 `start_mcp.sh` 原有模式
- 穩定、已驗證可用
- 延遲約 10-50ms（足夠大部分場景）

### 方案 2：混合模式（進階）

- Wi-Fi：感測器資料（影像、LiDAR）
- 有線：低延遲運動控制（直發 MCU）
- 需要額外開發整合

### 方案 3：升級到 Go2 EDU

- 完整的有線 ROS2 支援
- 內建 Jetson 運算
- 成本較高

從純雲端改成「**本地+雲端**」雙層架構：
