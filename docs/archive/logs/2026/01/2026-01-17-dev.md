# 2026-01-17 開發日記

## 今日進度
- 修復 `/scan` 全 `inf` 問題，已成功輸出實際距離值。
- `/point_cloud2` 由 `width=0` 修復為正常點數，`row_step` 正常。
- 建立 LSP/pyright 環境（`basedpyright` + `pyrightconfig.json` + `typings/lz4` stubs）。
- SLAM 參數調低（`map_update_interval`、`minimum_travel_distance/heading`），`range_min` 生效。
- `/odom` 在設定 `ROBOT_IP` 後開始正常發布，但 `/map` 仍大多為 `-1`。

## 問題與根因
- **/point_cloud2 為空（width=0、row_step=0）**
  - 根因：`update_meshes_for_cloud2()` 的 intensity 過濾把點雲全濾掉。
- **/scan 全 inf**
  - 根因 1：點雲為空。
  - 根因 2：`pointcloud_to_laserscan` 高度/距離參數過窄，容易濾光點雲。
- **/map 長時間為空**
  - 可能原因：SLAM 未持續接收 `/scan` 或 TF/里程計資料不完整，或參數門檻過高。
- **/slam_toolbox/feedback 無輸出**
  - 可能原因：未成功接收 `/scan` 或 TF 斷鏈。

## 解決方法
1. **強度過濾 fallback**
   - 檔案：`go2_robot_sdk/go2_robot_sdk/infrastructure/sensors/lidar_decoder.py`
   - 做法：當 intensity 過濾後為空，回退到未過濾點雲。

2. **pointcloud_to_laserscan 參數調整**
   - 檔案：`go2_robot_sdk/launch/robot.launch.py`
   - 調整（更新）：
     - `min_height: -0.25`
     - `max_height: 0.5`
     - `range_min: 0.2`
     - `range_max: 30.0`

3. **PointCloud2 frame 調整**
   - 檔案：`go2_robot_sdk/go2_robot_sdk/infrastructure/ros2/ros2_publisher.py`
   - `frame_id` 改為 `base_link`。

4. **解碼器路徑確認**
   - 檔案：`go2_robot_sdk/go2_robot_sdk/infrastructure/webrtc/data_decoder.py`
   - `deal_array_buffer()` 使用 LZ4 主路徑，失敗才回退 WASM。

5. **LSP/pyright 支援**
   - 新增 `pyrightconfig.json`（`stubPath: typings`）。
   - 新增 `typings/lz4/__init__.pyi` 與 `typings/lz4/block.pyi`。

6. **SLAM 參數調整**
   - 檔案：`go2_robot_sdk/config/mapper_params_online_async.yaml`
   - 調整：
     - `map_update_interval: 1.0`
     - `minimum_time_interval: 0.1`
     - `minimum_travel_distance: 0.1`
     - `minimum_travel_heading: 0.1`
