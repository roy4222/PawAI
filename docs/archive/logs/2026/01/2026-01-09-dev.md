## 📅 今日重點：專題重大轉向與 PawAI 品牌確立

今天進行了專案核心方向的深度調整。基於評審與教授的建議，我們決定放棄過於沉重的 YOLO-World + Depth Anything V2 遠端架構，轉而擁抱更具實踐性且具備前瞻性的 **VLA (Vision-Language-Action)** 架構。

---

## 🚀 一、專題方向大轉變：從「老人與狗」到「PawAI」

### 1. 核心定位調整
- **舊方案：** Depth Anything V2 + YOLO-World（依賴遠端 GPU，延遲高，應用受限）。
- **新方案：** `coco_detector` (FasterRCNN-MobileNet) + **RealSense D435** 深度攝影機。
- **品牌更名：** 從「老人與狗 (Elder and Dog)」改為「**PawAI: 基於多模態感知融合之自主尋物與具身互動**」。
- **避障邏輯：** 回歸導航本質。不再嘗試用 AI 解決避障，而是直接發揮 Nav2 + SLAM 的 SDK 原生能力。

### 2. 轉型理由
- **效能考量：** 遠端推論延遲難以克服，本地邊緣運算（System 1）應優先處理即時反應。
- **硬體升級：** 引入 RealSense D435 獲取穩定的物理深度資訊，解決單目測距的不穩定性。
- **學術高度：** 從單純的「尋物工具」提升至「具身智能 (Embodied AI)」的互動研究。

## 🧠 三、雙層 AI 架構設計提案 (Dual-System)

參考人類認知模型，我們提出「快慢系統」雙層架構：

### 1. 快系統 (System 1) - 本地 Mac VM
- **目標：** 低延遲 (<200ms)、高反應。
- **職責：** Nav2 + SLAM 導航、`coco_detector` 基礎感知、RealSense 深度處理、TTS 語音輸出、Safety Layer。

### 2. 慢系統 (System 2) - 學校 GPU Server (RTX 6000 Blackwell)
- **目標：** 高智能、深度理解。
- **職責：** 本地 LLM/VLM、Whisper STT、VLA 模型執行（處理複雜、長週期的決策）。

## 🔬 四、GPU Server 模型研究 (Librarian Agent 成果)

發現比原計畫（YOLO-World）更具競爭力的選擇，特別是 **VLA (Vision-Language-Action)** 模型：

### 1. VLA 模型：直接輸出動作 (Action Tokens)
| 模型 | VRAM | 特色 |
|------|------|------|
| **NORA 3B** | 8.3GB | 極輕量，基於 Qwen2.5-VL，適合邊緣部署。 |
| **OpenVLA 7B** | 10-12GB | 開源最強，超越 Google RT-2-X 16.5%。 |
| **MobileVLA-R1** | - | **已有 Unitree Go2 部署成功案例**。 |
| **NaVILA** | - | 四足導航專用，具備 88% 成功率。 |

### 2. 感知模型組合
- **SAM 2 (8-10GB)：** 44 FPS 實時實例分割。
- **RTMPose (2-4GB)：** **430+ FPS** 極速姿態估計（可用於人機互動）。
- **UniDepthV2：** 通用公制深度估計備案。

> **資源配置方案：** RTX 6000 (48GB) 可同時運行 `VLA (12GB) + LLM (16GB) + 感知 (10GB) + 姿態 (4GB) = 42GB` ✅。
