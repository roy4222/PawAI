## 📅 今日重點：專題發表與評審回饋

### 🎤 評審會議重點整理

今天進行了第一階段專題發表，評審老師提出了四大維度的核心建議：

> **核心觀點：不要用 AI 去做傳統技術已經解掉的事情，而應發揮 LLM 在邏輯理解與模組化擴展上的長處。**

---

## 🔧 一、技術架構與效能優化

### 問題：避障邏輯延遲高

**現狀：**
- 利用「拍照 → 傳給 LLM → 回傳指令」的片段式執行
- 體感延遲（Latency）極高，機器狗動作不連貫
- 像「登陸火星」一樣走走停停

**老師建議：**
- 避障（從 A 點到 B 點）不應交給 LLM 處理
- 這是六、七年前就已被深度相機與傳統導航算法解決的問題
- 應利用深度影像的連續流，設定障礙物距離閾值，讓機器狗自動尋路

### 問題：網路依賴

**現狀：** 依賴學校雲端 GPU 伺服器，受限於網路帶寬與干擾

**老師建議：**
- 應嘗試本地化部署（Edge AI）
- 即便準確率從 99% 降到 70-80% 也沒關係，只要能換取即時反應
- 建議尋找參數量小的模型（如 Jetson Nano）

### 問題：鏡頭校正

**現狀：** 僅用一個校正係數處理廣角鏡頭深度

**老師建議：**
- 單張圖換算深度在物理上是不穩定的
- 建議更換或增加深度攝影機（Depth Camera），直接拿深度圖資料

## 🎯 二、應用場景與目標設定

### 問題：物件尺寸限制

**現狀：** 居家老人最常弄丟的是手機、鑰匙、藥包等小東西

**老師建議：**
- 機器狗高度有限，視覺方案對小東西的辨識與測距精度可能不足
- 需明確定義機器狗能找的物體限制
- 或者改進視角與演算法以符合真實需求

### 問題：環境強健性

**老師建議：**
- 光照變化、家具移動、物體被遮擋一半的情況非常普遍
- 目前系統在遮擋下會找不到目標
- 應提升系統在複雜環境下的「強健性」

---

## 🛡️ 三、系統安全性與防護層

### 問題：LLM 直接控制底層

**現狀：** 讓 LLM 直接透過 MCP 控制 ROS2 底層極其危險

**老師建議：**
- 如果 AI 理解錯誤或下達錯誤指令，可能會導致機器狗絆倒長者或損壞

**解決策略：**

1. **確認機制：** 執行重大動作前，由機器狗「說出」它打算做什麼
   - 例：「我看到椅子在右邊，我現在要轉過去囉」
   - 讓使用者有確認或喊停的機會

2. **邏輯防線：** 在機器狗端設定物理極限（速度、轉角限制）
   - 無論 AI 下什麼指令，都不能突破安全界線

---

## 🚀 四、未來創意與亮點發揮

### Skills 模組化

**老師建議：**
- LLM 的強項在於處理模糊指令（如「拿桌上的水杯過來給我」）
- 應將重點放在 AI 如何分析並轉化為多個步驟的執行
- 將機器狗的能力打包成 Skill（如：避障、尋物、跳舞、陪伴提醒）
- 透過 MCP 協議，讓 AI 像呼叫 API 一樣靈活運用

### IoT 整合

**老師建議：**
- 既然針對老人，可以結合智慧手環（IoT 數據）
- 當手環測到心跳異常或跌倒時，機器狗應主動前往查看並傳回即時畫面

## 🔬 Go2 技術深度研究與連線實測

### 1. Go2 Pro 核心硬體與連線能力

> ⚠️ **重要修正：** 經過實測連線，確認 Go2 Pro 版本**無內建 Jetson (192.168.123.18)**。

| 設備 IP | 功能名稱 | 連通性 (Pro) | 備註 |
|----|------|---------|---------|
| `192.168.123.161` | **MCU (主控底層)** | ✅ 可連通 | 處理運動控制與低階指令 |
| `192.168.123.18` | **Jetson NX/Orin** | ❌ 無回應 | 僅 EDU 版本配備 |
| `192.168.123.20` | **LiDAR** | ⚠️ 待激活 | 透過 SLAM 模式呼叫 |

**實測連線環境設定 (Windows 直連 Go2)：**
- **電腦 IP:** `192.168.123.99`
- **子網路遮罩:** `255.255.255.0`
- **閘道:** 留空
- **工具:** PowerShell `ping 192.168.123.161` (延遲 <1ms)

### 2. go2_ros2_sdk 分支研究與有線模式

確認 `go2_robot_sdk` 支援 **CycloneDDS (Ethernet)** 模式，可解決 Wi-Fi 延遲問題。

*   **優勢：** 實現 <1ms 的極低控制延遲。
*   **關鍵腳本：** 已建立 `start_go2_wired.sh` 以簡化連線流程。
*   **配置方式：**
    ```bash
    export CONN_TYPE="cyclonedds"
    export ROBOT_IP="192.168.123.161" # 指向 MCU 地址
    export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
    ```

### 3. 本地運算與逆向工程發現

*   **CPU 運算：** Go2 Pro 搭載 8 核 CPU，理論上可運行 **YOLOv8n** 輕量模型 (約 10-20 FPS)，雖無 Jetson 算力強，但仍具備邊緣辨識潛力。
*   **社群突破：** 參考 Reddit 逆向工程討論，Pro/Air 型號可透過連入內部 Rockchip SoC 嘗試解鎖 EDU 功能。
*   **控制協議：** 遙控器使用 **SBUS 協議**，未來可考慮透過物理/電性改裝達成更高權限的接管。
