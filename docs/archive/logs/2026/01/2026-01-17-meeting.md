### NVIDIA Jetson 闆子選擇

| 型號 | VRAM | 價格 | 備註 |
|------|------|------|------|
| **Jetson Orin Nano 8GB** | 8GB | ~$14,800 | 遠端 GPU + 本地推論 |
| **Jetson Orin Nano 16GB** | 16GB | ~$28,000+ | 全本地運算 |

**為什麼需要闆子？**
1. Go2 Pro 無外接孔 → 無法接深度攝影機
2. 有線開發超麻煩 → 闆子可走 WiFi
3. 本地運算 → 減少網路延遲
4. 參考架構：闆子背在狗背上 + 深度攝影機在前

**決策：** 先測試 8GB 模型是否夠用，再決定購買規格

## 🎯 關鍵決策與共識

### 1. 架構確認

```
┌─────────────────────────────────────────────────────────┐
│                    GPU Server (雲端)                     │
│         VLA/VLM (決策)、大型 LLM (複雜推理)              │
└──────────────────────────┬──────────────────────────────┘
                           │ HTTP API
                           ↓
┌─────────────────────────────────────────────────────────┐
│              Jetson 闆子 (背在狗上)                      │
│    深度攝影機處理、coco_detector、STT、輕量 LLM          │
└──────────────────────────┬──────────────────────────────┘
                           │ WiFi / 有線
                           ↓
┌─────────────────────────────────────────────────────────┐
│                    Go2 Pro (MCU)                        │
│           動作執行 (SDK 只傳指令)、防摔、B 站            │
└─────────────────────────────────────────────────────────┘
```

### 2. SDK 定位調整

- **原本想法：** 用 SDK 的 SLAM/Nav2
- **現實：** Go2 Pro 效能不足，SDK 資料傳輸有瓶頸
- **新策略：** SDK 只負責「傳動作指令」，其他全在闆子/雲端

### 3. 參考對象

- 黃懷生老師 6 年前專題（TurtleBot）
- 功能類似：導航、避障、物件偵測、人臉辨識
- 我們優勢：更強機體 + LLM 加持
- 目標：「把基礎功能做出來，再疊加 LLM 創新」
