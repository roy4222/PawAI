## 🔴 P0-1：ROS2 核心概念（40 分鐘）

### 1.1 什麼是 ROS2？

**ROS2 (Robot Operating System 2)** 是一個機器人開發框架，提供：
- 模組化的節點架構
- 標準化的通訊協定
- 豐富的工具生態系統

**全端類比：** 把 ROS2 想像成「機器人界的 Kubernetes + 訊息佇列」

```
Web 開發                      ROS2
─────────────────────────────────────────────
微服務 (Microservice)    ←→   節點 (Node)
Kafka/RabbitMQ           ←→   Topic (發布/訂閱)
REST API                 ←→   Service (請求/回應)
WebSocket + 長連接       ←→   Action (非同步任務)
JSON Schema              ←→   Message (訊息定義)
Docker Container         ←→   Package (套件)
```

---

### 1.2 四大通訊機制

#### 📤 Topic（發布/訂閱模式）

**概念：** 多對多的非同步通訊，類似 Message Queue

```python
# 發布者 (Publisher) - 類似 Kafka Producer
class VelocityPublisher(Node):
    def __init__(self):
        super().__init__('velocity_publisher')
        # 建立發布者，發布到 /cmd_vel topic
        self.publisher = self.create_publisher(Twist, '/cmd_vel', 10)
        
    def send_velocity(self, linear_x, angular_z):
        msg = Twist()
        msg.linear.x = linear_x    # 前進速度 (m/s)
        msg.angular.z = angular_z  # 旋轉速度 (rad/s)
        self.publisher.publish(msg)

# 訂閱者 (Subscriber) - 類似 Kafka Consumer
class VelocitySubscriber(Node):
    def __init__(self):
        super().__init__('velocity_subscriber')
        # 建立訂閱者，訂閱 /cmd_vel topic
        self.subscription = self.create_subscription(
            Twist,
            '/cmd_vel',
            self.velocity_callback,
            10
        )
    
    def velocity_callback(self, msg):
        self.get_logger().info(f'收到速度指令: x={msg.linear.x}, z={msg.angular.z}')
```

**本專案常用的 Topics：**

| Topic 名稱          | 訊息類型                 | 方向 | 說明            |
| ------------------- | ------------------------ | ---- | --------------- |
| `/cmd_vel`          | `geometry_msgs/Twist`    | 輸出 | 速度控制指令    |
| `/scan`             | `sensor_msgs/LaserScan`  | 輸入 | LiDAR 掃描資料  |
| `/camera/image_raw` | `sensor_msgs/Image`      | 輸入 | 相機原始影像    |
| `/odom`             | `nav_msgs/Odometry`      | 輸入 | 里程計資料      |
| `/tf`               | `tf2_msgs/TFMessage`     | 輸入 | 座標轉換資訊    |
| `/map`              | `nav_msgs/OccupancyGrid` | 輸入 | SLAM 建立的地圖 |

#### 🔄 Service（請求/回應模式）

**概念：** 一對一的同步通訊，類似 REST API

```python
# Service 伺服器 - 類似 Express.js 的 route handler
class SnapshotService(Node):
    def __init__(self):
        super().__init__('snapshot_service')
        # 建立服務，類似 app.get('/capture_snapshot', ...)
        self.srv = self.create_service(
            Trigger,                    # 服務類型 (類似 API Schema)
            '/capture_snapshot',        # 服務名稱 (類似 endpoint)
            self.capture_callback       # 處理函數 (類似 handler)
        )
    
    def capture_callback(self, request, response):
        # 處理請求，回傳結果
        response.success = True
        response.message = "base64_encoded_image..."
        return response

# Service 客戶端 - 類似 fetch() 或 axios
class SnapshotClient(Node):
    def __init__(self):
        super().__init__('snapshot_client')
        self.client = self.create_client(Trigger, '/capture_snapshot')
        
    async def capture(self):
        request = Trigger.Request()
        future = self.client.call_async(request)
        response = await future
        return response.message
```

**本專案用到的 Services：**

| Service 名稱           | 類型                | 說明                       |
| ---------------------- | ------------------- | -------------------------- |
| `/capture_snapshot`    | `std_srvs/Trigger`  | 擷取相機影像（我們開發的） |
| `/map_server/save_map` | `nav2_msgs/SaveMap` | 儲存 SLAM 地圖             |

#### ⏳ Action（非同步任務模式）

**概念：** 長時間任務 + 即時回饋 + 可取消，類似 WebSocket + 進度條

```
Client                          Server
  │                                │
  │─── Goal Request ──────────────>│  「導航到 (1.0, 0.0)」
  │<── Goal Accepted ──────────────│  「收到，開始執行」
  │                                │
  │<── Feedback ───────────────────│  「距離剩餘 0.8m」
  │<── Feedback ───────────────────│  「距離剩餘 0.5m」
  │<── Feedback ───────────────────│  「距離剩餘 0.2m」
  │                                │
  │<── Result ─────────────────────│  「導航完成！」
  │                                │
```

**Python 範例：**

```python
from nav2_msgs.action import NavigateToPose
from rclpy.action import ActionClient

class NavigationClient(Node):
    def __init__(self):
        super().__init__('navigation_client')
        # 建立 Action 客戶端
        self.action_client = ActionClient(
            self,
            NavigateToPose,
            'navigate_to_pose'
        )
    
    def send_goal(self, x, y):
        goal_msg = NavigateToPose.Goal()
        goal_msg.pose.header.frame_id = 'map'
        goal_msg.pose.pose.position.x = x
        goal_msg.pose.pose.position.y = y
        goal_msg.pose.pose.orientation.w = 1.0
        
        # 發送目標，註冊 feedback callback
        self.action_client.send_goal_async(
            goal_msg,
            feedback_callback=self.feedback_callback
        )
    
    def feedback_callback(self, feedback_msg):
        feedback = feedback_msg.feedback
        distance = feedback.distance_remaining
        self.get_logger().info(f'距離剩餘: {distance:.2f}m')
```

**本專案用到的 Actions：**

| Action 名稱         | 類型                       | 說明              |
| ------------------- | -------------------------- | ----------------- |
| `/navigate_to_pose` | `nav2_msgs/NavigateToPose` | Nav2 導航到指定點 |
| `/follow_path`      | `nav2_msgs/FollowPath`     | 沿路徑移動        |
| `/spin`             | `nav2_msgs/Spin`           | 原地旋轉          |

**概念：** 類似 TypeScript 的 interface 或 JSON Schema

**常用訊息類型：**

```python
# geometry_msgs/Twist - 速度指令
msg = Twist()
msg.linear.x = 0.5   # 前進 0.5 m/s
msg.linear.y = 0.0   # 側移 (差動機器人通常為 0)
msg.linear.z = 0.0   # 垂直 (地面機器人為 0)

msg.angular.y = 0.0  # 滾轉 (通常為 0)
msg.angular.z = 0.3  # 旋轉 0.3 rad/s (左轉)

# geometry_msgs/PoseStamped - 位置姿態（含時間戳記與座標框架）
pose = PoseStamped()
pose.header.frame_id = 'map'           # 座標框架
pose.header.stamp = self.get_clock().now().to_msg()  # 時間戳記
pose.pose.position.x = 1.0             # X 座標
pose.pose.position.y = 2.0             # Y 座標
pose.pose.position.z = 0.0             # Z 座標 (地面為 0)
pose.pose.orientation.w = 1.0          # 四元數 (朝向正前方)

# sensor_msgs/LaserScan - LiDAR 掃描
# ranges[0] = 正前方距離
# ranges[90] = 左方距離 (假設 360° LiDAR)
# ranges[180] = 正後方距離
# ranges[270] = 右方距離
```

### 1.3 QoS（服務品質）

**為什麼重要：** 你在 12/7 遇到的 QoS 不相容問題就是這個！

```python
from rclpy.qos import QoSProfile, QoSReliabilityPolicy, QoSHistoryPolicy

# 預設 QoS（可靠傳輸）
default_qos = QoSProfile(depth=10)

# Best Effort QoS（即時但可能丟失）
# Go2 Driver 使用這個！
best_effort_qos = QoSProfile(
    reliability=QoSReliabilityPolicy.BEST_EFFORT,  # 不重傳
    history=QoSHistoryPolicy.KEEP_LAST,
    depth=1
)

# 訂閱時需匹配 Publisher 的 QoS
self.create_subscription(
    Image,
    '/camera/image_raw',
    self.callback,
    best_effort_qos  # ← 必須用 BEST_EFFORT 才能收到 Go2 的相機影像！
)
```

**QoS 相容性規則：**

| Publisher   | Subscriber  | 結果       |
| ----------- | ----------- | ---------- |
| RELIABLE    | RELIABLE    | ✅ 可以通訊 |
| RELIABLE    | BEST_EFFORT | ✅ 可以通訊 |
| BEST_EFFORT | RELIABLE    | ❌ 不相容！ |
| BEST_EFFORT | BEST_EFFORT | ✅ 可以通訊 |

## 🔴 P0-2：TF2 座標系統（30 分鐘）⭐⭐⭐ 超重要！

### 2.1 為什麼需要 TF？

想像你在室內導航：
- 「水杯在**桌子上方** 30cm」→ 這是相對於桌子的座標
- 「水杯在**房間座標** (2.5, 3.0)」→ 這是世界座標

機器人也一樣：
- 相機看到物體在**畫面中央** → 相機座標
- 需要轉換成**房間裡的位置** → 世界座標 → 才能導航過去

**TF2 = Transform (座標轉換) 系統**

---

### 2.2 TF 樹結構

```
                    map
                     │
                     │ ← SLAM/AMCL 提供（機器狗在地圖上的位置）
                     ↓
                    odom
                     │
                     │ ← 里程計提供（相對移動）
                     ↓
                 base_link ─────────────────────┐
                     │                          │
        ┌────────────┼────────────┐             │
        ↓            ↓            ↓             ↓
  front_camera   lidar_link   imu_link   [其他關節]
   (相機)         (LiDAR)      (IMU)
```

**座標框架說明：**

| 框架名稱       | 說明                     | 誰提供    |
| -------------- | ------------------------ | --------- |
| `map`          | 世界座標（固定不動）     | SLAM/AMCL |
| `odom`         | 里程計座標（會累積誤差） | 里程計    |
| `base_link`    | 機器狗本體中心           | URDF      |
| `front_camera` | 相機座標                 | URDF      |
| `lidar_link`   | LiDAR 座標               | URDF      |

### 2.3 座標轉換概念

```python
import tf2_ros
from geometry_msgs.msg import PointStamped

class CoordinateTransformer(Node):
    def __init__(self):
        super().__init__('coord_transformer')
        
        # TF2 Buffer 和 Listener
        self.tf_buffer = tf2_ros.Buffer()
        self.tf_listener = tf2_ros.TransformListener(self.tf_buffer, self)
    
    def camera_to_map(self, camera_point):
        """將相機座標系的點轉換到地圖座標系"""
        
        # 建立帶座標框架的點
        point_camera = PointStamped()
        point_camera.header.frame_id = 'front_camera'  # ⚠️ 不是 camera_link！
        point_camera.header.stamp = self.get_clock().now().to_msg()
        point_camera.point.x = camera_point[0]
        point_camera.point.y = camera_point[1]
        point_camera.point.z = camera_point[2]
        
        try:
            # 查詢轉換
            transform = self.tf_buffer.lookup_transform(
                'map',           # 目標座標系
                'front_camera',  # 來源座標系
                rclpy.time.Time()
            )
            
            # 執行轉換
            point_map = tf2_geometry_msgs.do_transform_point(
                point_camera, transform
            )
            
            return (point_map.point.x, point_map.point.y, point_map.point.z)
            
        except tf2_ros.TransformException as e:
            self.get_logger().error(f'TF 轉換失敗: {e}')
            return None
```

### 2.5 Static TF vs Dynamic TF

| 類型           | 說明           | 範例                   |
| -------------- | -------------- | ---------------------- |
| **Static TF**  | 固定不變的轉換 | 相機裝在機器狗哪個位置 |
| **Dynamic TF** | 會隨時間改變   | 機器狗在地圖上的位置   |

```python
# 發布 Static TF（通常在 launch 檔案或 URDF 定義）
from tf2_ros import StaticTransformBroadcaster

broadcaster = StaticTransformBroadcaster(self)
static_transform = TransformStamped()
static_transform.header.frame_id = 'base_link'
static_transform.child_frame_id = 'my_sensor'
static_transform.transform.translation.x = 0.1  # 感測器在前方 10cm
broadcaster.sendTransform(static_transform)

# Dynamic TF 由 SLAM/Odom 節點自動發布
```

## 🟡 P1-1：Nav2 導航框架（25 分鐘）

### 3.1 Nav2 是什麼？

**Nav2 (Navigation 2)** 是 ROS2 的標準導航套件，提供：
- 路徑規劃（從 A 到 B 怎麼走）

### 3.3 Costmap（成本地圖）

**概念：** 把環境轉換成「難度分數」的格子地圖

```
255 = 絕對障礙（牆壁、物體）
254 = 必撞區（太近了）
253 = 內部膨脹區
128 = 中等風險區
  1 = 略有風險
  0 = 自由空間（可自由通行）
```

```yaml
      plugin: "dwb_core::DWBLocalPlanner"
      max_vel_x: 0.5                    # 最大前進速度 (m/s)
      min_vel_x: 0.1                    # 最小前進速度 ← 避免原地打轉！
      max_vel_theta: 1.0                # 最大旋轉速度 (rad/s) ← 降低可防止瘋狂旋轉
      min_vel_theta: 0.0
      acc_lim_x: 2.5                    # 前進加速度限制
      acc_lim_theta: 3.2                # 旋轉加速度限制

local_costmap:
  local_costmap:
    ros__parameters:
      robot_radius: 0.3                 # 機器狗半徑 (m)
      resolution: 0.05                  # 地圖解析度 (m/格)
      
      inflation_layer:
        inflation_radius: 0.5           # 膨脹半徑（安全距離）

global_costmap:
  global_costmap:
    ros__parameters:
      global_frame: map                 # ⚠️ 必須是 map，不是 odom！
      robot_base_frame: base_link
```

## 🟡 P1-2：MCP 協定概念（15 分鐘）

### 4.1 什麼是 MCP？

**MCP (Model Context Protocol)** 是 Anthropic 提出的 LLM 與外部工具通訊標準。

**傳統方式 vs MCP：**

```
❌ 傳統方式：
LLM → 生成 Python 程式碼 → 執行 → 取得結果

✅ MCP 方式：
LLM → 直接呼叫工具 → 取得結果
```

**類比：**
```
Web 開發                      MCP
─────────────────────────────────────────────
REST API Server          ←→   MCP Server
API Endpoints            ←→   Tools (工具)
API Client (fetch)       ←→   MCP Client (LLM)
JSON Response            ←→   Tool Result
```

---

### 4.2 ros-mcp-server 提供的工具

本專案使用 [ros-mcp-server](https://github.com/nicholasgubbins/ros-mcp-server) 讓 LLM 控制 ROS2。

**可用工具：**

| 工具名稱           | 說明            | 範例                                                     |
| ------------------ | --------------- | -------------------------------------------------------- |
| `get_topics`       | 列出所有 Topics | `get_topics()`                                           |
| `get_topic_info`   | 取得 Topic 資訊 | `get_topic_info('/cmd_vel')`                             |
| `subscribe_once`   | 訂閱一次訊息    | `subscribe_once('/scan', 'sensor_msgs/LaserScan')`       |
| `publish_once`     | 發布一次訊息    | `publish_once('/cmd_vel', 'geometry_msgs/Twist', {...})` |
| `call_service`     | 呼叫 Service    | `call_service('/capture_snapshot', 'std_srvs/Trigger')`  |
| `send_action_goal` | 發送 Action     | `send_action_goal('/navigate_to_pose', ...)`             |

---

### 4.3 實際使用範例

**在 Claude Desktop 或 Kilo Code 中：**

```
使用者：「讓機器狗往前走 1 公尺」

LLM 思考：
1. 我需要發送速度指令到 /cmd_vel
2. 前進 1 公尺，假設速度 0.3 m/s，需要約 3.3 秒
3. 使用 publish_once 發送指令

LLM 呼叫：
publish_once(
    topic_name="/cmd_vel",
    message_type="geometry_msgs/Twist",
    message={
        "linear": {"x": 0.3, "y": 0.0, "z": 0.0},
        "angular": {"x": 0.0, "y": 0.0, "z": 0.0}
    }
)
```

**導航範例：**

```
使用者：「導航到座標 (1.0, 0.5)」

LLM 呼叫：
send_action_goal(
    action_name="/navigate_to_pose",
    action_type="nav2_msgs/action/NavigateToPose",
    goal={
        "pose": {
            "header": {"frame_id": "map"},
            "pose": {
                "position": {"x": 1.0, "y": 0.5, "z": 0.0},
                "orientation": {"w": 1.0}
            }
        }
    },
    timeout=60.0
)
```

## 🟢 P2-3：CycloneDDS 網路配置實戰（15 分鐘）⚠️ 必讀！

### 7.1 為什麼需要特別關注這個？

**你在 12/01 和 12/07 遇到的問題根源就是這個！**

```
常見症狀：
❌ ros2 node list 看得到節點，但 ros2 topic echo 收不到資料
❌ ros2 topic hz 顯示 "no data received"
❌ Terminal A 可以發布，但 Terminal B 收不到
```

---

### 7.2 本專案的網路拓樸

```
┌─────────────────────────────────────────────────────────────┐
│                     你的 Ubuntu VM                           │
├─────────────────────────────────────────────────────────────┤
│  網卡 1: lo (127.0.0.1)         ← ROS2 本機通訊用（最重要！）│
│  網卡 2: enp0s1 (192.168.12.222) ← 連接 Go2 機器狗          │
│  網卡 3: enp0s2 (192.168.1.xxx)  ← 連接家用網路/Windows     │
└─────────────────────────────────────────────────────────────┘
```

**核心問題：** DDS 預設會嘗試所有網卡，導致封包走錯路徑！

---

### 7.3 解決方案：local_only_v2.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CycloneDDS xmlns="https://cdds.io/config">
    <Domain>
        <General>
            <NetworkInterfaceAddress>127.0.0.1</NetworkInterfaceAddress>
            <!-- 強制只走 loopback，避免多網卡干擾 -->
        </General>
    </Domain>
</CycloneDDS>
```

**為什麼這樣設定：**
- ✅ VM 內所有 ROS2 節點都在同一台機器上
- ✅ 不需要跨網路通訊（Go2 Driver 透過 WebRTC，不是 ROS2 DDS）
- ✅ loopback 速度最快、最穩定

```bash
# Step 3: 檢查 TF 樹
ros2 run tf2_tools view_frames
# ❌ 如果 TF 樹是空的 → SLAM 沒有運行

# Step 4: 檢查環境變數
echo $CYCLONEDDS_URI
# ❌ 如果是空的 → 重新載入環境
```

## 🎯 附錄：本專案的實戰問題速查表

### 問題 1：機器狗不動

**症狀：** 發送 `/cmd_vel` 但機器狗沒反應

```bash
# 診斷流程
ros2 topic echo /cmd_vel  # 確認有訊息
→ 有訊息但不動 → 檢查 Go2 Driver
→ 沒訊息 → 檢查 Nav2 或 MCP

ros2 topic hz /scan  # 確認 Driver 運行
→ Timeout → WebRTC 連線失敗（重啟 Go2）
→ 正常 → Driver 正常
```

---

### 問題 2：Nav2 原地打轉

**症狀：** 導航目標發送後，機器狗只旋轉不前進

**原因：** `min_vel_x = 0.0`（允許零速前進）

**解決：** 修改 `nav2_params.yaml`
```yaml
controller_server:
  ros__parameters:
    FollowPath:
      min_vel_x: 0.1  # 強制前進
```

---

### 問題 3：看得到節點但收不到資料

**症狀：**
```bash
ros2 node list  # ✅ 看得到節點
ros2 topic hz /scan  # ❌ Timeout
```

**原因：** CycloneDDS 配置錯誤或未設定

**解決：**
```bash
export CYCLONEDDS_URI=/home/roy422/local_only_v2.xml
ros2 daemon stop && ros2 daemon start
```

### 問題 7：QoS 不相容警告

**症狀：**
```
[WARN] New publisher discovered on topic '/camera/image_raw',
offering incompatible QoS. Last incompatible policy: RELIABILITY
```

**原因：** Subscriber 使用 `RELIABLE`，但 Publisher 是 `BEST_EFFORT`

**解決：** 使用 `BEST_EFFORT` QoS
```python
from rclpy.qos import QoSProfile, QoSReliabilityPolicy

best_effort_qos = QoSProfile(
    reliability=QoSReliabilityPolicy.BEST_EFFORT,
    depth=1
)
self.create_subscription(Image, '/camera/image_raw', callback, best_effort_qos)
```

### 問題 10：snapshot_service 沒註冊

**症狀：**
```bash
ros2 run go2_robot_sdk snapshot_service
# Package 'go2_robot_sdk' not found
```

**原因：** 沒在 `setup.py` 註冊 entry point

**解決：** 編輯 `setup.py`

```python
entry_points={
    'console_scripts': [
        'snapshot_service = go2_robot_sdk.snapshot_service:main',
    ],
},
```

然後重建：
```bash
colcon build --packages-select go2_robot_sdk
source install/setup.zsh
```
