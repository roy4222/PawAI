## 📅 階段性任務詳解

### 🚨 Phase 1：連線突破戰 (11/23 - 11/26)

**目標：** 不管用什麼手段，要在你們家裡的 Windows 螢幕上看到 GPU 伺服器的桌面。

1. **基礎連線 (Basic)**
    - [ ]  確認 SSH 可連入 GPU 伺服器。
    - [ ]  執行 `nvidia-smi`，確認 RTX 8000 運作正常。
    - [ ]  執行 `python3 -m http.server 8000`，測試家裡瀏覽器能否連上 (測試 Port 轉發)。
2. **畫面傳輸 (Streaming) - 依序嘗試**
    - **方案 A (官方串流)**：
        - 在 Server 跑 `./run_sim.sh --headless --streaming native`。
        - 在 Windows 開 Omniverse Streaming Client 連線。
    - **方案 B (遠端桌面 - 強烈推薦備案)**：
        - 若方案 A 失敗，立刻安裝 **NoMachine** (首選) 或 **TigerVNC**。
        - **優點**：穿透力強，學校防火牆通常擋不住。
    - **方案 C (虛擬內網 - 最後手段)**：
        - 若連 IP 都 Ping 不到，安裝 **ZeroTier** 或 **Tailscale** 打洞。

**📅 11/26 (週二) 驗收點：**

> 必須回報：「我能用 [方式] 看到 GPU 伺服器的桌面了。」

### 🔗 Phase 3：AI API 並行開發 (12/01 - 12/10)

**目標：** 部署 C 組的 VLM 模型，提供 HTTP API 給 Roy 呼叫。

1. **資源調度 (Resource Management)**
    - RTX 8000 有 48GB VRAM。
    - Isaac Sim 吃 ~15-20GB。
    - VLM 模型吃 ~10-15GB。
    - **結論**：**可以同時跑！** 但為了穩定，請準備好切換腳本。
2. **撰寫控制腳本**
    - `start_sim.sh`：殺掉 AI，啟動模擬器。
    - `start_vlm.sh`：殺掉模擬器，啟動 AI API。
3. **API 部署**
    - [ ]  部署 `inference_server.py` (FastAPI)。
    - [ ]  開放 Port 8000。
    - [ ]  **整合測試**：Roy 發送一張 `test.jpg`，伺服器回傳座標 JSON。
