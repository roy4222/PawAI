## 2. 衝突點（必須做出決定，不能裝沒看到）

### 2.1 🚨 主場景：導盲犬 vs 機構巡檢

| 場次 | 第一推薦 | 對「老人陪伴」 |
|---|---|---|
| 葉 5/18 | 像狗的互動（招手 / 跟隨）；質疑老人居家陪伴**安全風險** | ⚠️ 強烈質疑（20kg 拉手摔倒 / 骨折） |
| 雅文 5/20 | **導盲犬 POC**（容錯率高、補盲區、移動性發揮） | ❌ 質疑「老人容錯率也不一定高」 |
| 呂奇傑 5/22 | 長照中心交誼廳串連（接受老人，「老人的學生宿舍」） | ✅ 接受 |
| **v2 spec 5/22 amend** | **機構巡檢（長照 / 日照中心）** | △ 條件接受（不取代照護人員、POC 框、不直接接觸） |

**衝突**：
- 雅文第一推薦的**導盲犬**在 v2 spec 被放到 §12 L3（1 年+）
- 葉的**安全顧慮**（拉手摔倒）對機構場景同樣存在

**決議建議**（待 5/26 sprint kickoff 確認）：

維持 v2 spec 的「**機構巡檢主場景**」決定，理由：
1. 教授 5/21 pivot 已 commit + 呂奇傑 5/22 強烈支持
2. 導盲犬需要真實戶外導航 + 高安全驗證，6/18 不可達；機構走廊單純、可控
3. 機構巡檢已自然吸收雅文的「巡檢 / 火場」次推薦邏輯（人不能進、實體巡檢）

**但必須對雅文回應**：~~把「導盲 / 弱勢輔助」放未來情境第一個~~ **此建議於 2026-05-23 grill Q3 後改為維持 v2 spec 原順序（校園 → 導盲 → 救災）**。對雅文回應改用其他方式（例如 sprint kickoff 主動報告主場景採呂奇傑路線的理由 + 5/27 影片旁白明確點出「導盲是雅文老師建議的未來延伸方向」）。

### 2.2 🚨 老人直接互動的安全聲明

葉 5/18 警示在 v2 spec 沒被認真處理：
> 「20 公斤、走路蹦蹦響、拉手就摔倒、撞到老人腿可能斷」

**決議**：
- Demo 影片**不能出現**「老人摸 PawAI / 拉手 / 抱 PawAI / 跟 PawAI 玩」鏡頭
- 改成**「PawAI 在合理距離（≥ 1m）報告 / 觀察 / 點名」**，不直接物理接觸
- 文件 + 影片旁白必須主動聲明：
  > 「PawAI 設計為**非接觸式機構巡檢助理**，不替代照護人員的直接照顧。一切物理協助由照護人員執行，PawAI 只負責感知與回報。」
- v2 spec §12「非目標」加一條：**「不做直接物理接觸長者的互動鏡頭」**

### 2.3 文件結構（葉獨有，需立即修）

葉 5/18 明確說：
- 文件 40 頁太多，**刪規格表、刪準確率章節**
- 架構圖優先
- 不要硬套 User Story 格式
- **整合派不該談準確率**（用現成模型不是自己訓練）

**衝突**：v2 spec 沒涵蓋文件格式。
**決議**：6/18 文件版本必須遵守葉建議：
1. 刪除所有準確率 / mAP / FPS 章節（除非是 demo readiness 上下文）
2. 每個模組以「**用了什麼 + 做什麼用 + 踩了什麼坑**」三段式，規格表壓在附錄
3. 架構圖優先（v2 spec §3 任務閉環圖適合放正文最前）
4. NumPy 衝突、LiDAR 三顆電池、降壓板、SDK 破解 → **獨立「整合挑戰」章節**

## 4. 對外敘事禁用詞 / 必說句清單

### 4.1 ❌ 禁用詞（demo 影片旁白、簡報、口頭問答 都禁）

- 「本來想做 X」
- 「現在還沒做到」
- 「比較差 / 不完整 / 不穩定」
- 「期待不要太高」
- 「玩具」「寵物」（雅文已用「玩具」貶過 PAI）
- 「居家陪伴」（已被多位質疑）
- 「比 GPU 慢」「跑得不如人家好」

### 4.2 ✅ 必說句（每場 demo 都要出現）

- 「**沒有 PawAI 之前，Go2 只能用遙控器；有了 PawAI 之後，照護人員可以用自然語言派它巡檢，並收到結構化回報。**」
- 「**這是 Physical AI 在機構照護場域的原型驗證（POC），不取代照護人員。**」
- 「**這隻狗的不可替代性是：能移動、有實體、多模態感知三件事同時做到。手機做不到、固定攝影機做不到、雲端 AI 也做不到。**」
- 「**為了讓家用版 Go2 變成可程式控制的具身平台，我們做了 [3D 列印 / 降壓板 / SDK 破解 / 雙系統有線整合 / NumPy 版本治理]。**」

### 4.3 安全聲明（必說，責任歸屬問題）

- 「**PawAI 設計為非接觸式機構巡檢助理。**」
- 「**所有物理協助仍由照護人員執行，PawAI 只負責感知、回報、提醒。**」
- 「**安全機制示例：請 PawAI 翻跟斗時系統會拒絕執行**」（demo 親自跑這條）
