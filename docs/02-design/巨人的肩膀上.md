## 📦 核心依賴專案

### 1. Go2 WebRTC 通訊層

#### tfoldi/go2-webrtc

**GitHub:** https://github.com/tfoldi/go2-webrtc

**貢獻：**
- 🔓 **首次破解 Unitree Go2 的 WebRTC 協定**
- 研究 Go2 機器狗的網路通訊方式
- 實現 DTLS 加密握手
- 建立基礎的連線和資料接收機制

**技術亮點：**
- 逆向工程 Unitree 的私有協定
- 分析 WebRTC DataChannel 的資料格式
- 實現加密通訊

### 2. ROS2 生態系

#### ros-mcp-server (robotmcp)

**GitHub:** https://github.com/robotmcp/ros-mcp-server
**作者:** robotmcp 團隊
**授權:** Apache 2.0

**貢獻：**
- 🔌 **實現 MCP (Model Context Protocol) 與 ROS/ROS2 的橋接**
- 讓 LLM (Claude/GPT/Gemini) 可以直接控制 ROS2 機器人
- 支援 Topics、Services、Parameters，Actions 支援開發中
- 雙向通訊：AI 不僅能控制，還能即時觀察機器人狀態

**技術亮點：**
- 透過 `rosbridge_server` (WebSocket) 與 ROS2 通訊
- 支援 ROS1 與 ROS2，無需修改機器人原始碼
- MCP 標準協定，相容所有支援 MCP 的 LLM 客戶端
- 完整的型別系統支援（包含自訂 msg/srv）

**對本專案的意義：**
- 🌟 **這是「老人與狗」專題轉型 MCP 架構的核心元件**
- 讓 AI 能夠「聽懂人話」並控制機器狗
- 實現「爺爺說『幫我找水』→ Claude 理解 → Go2 執行」的完整鏈路
- 提供視覺閉環：AI 能讀取相機快照，動態調整導航策略

**實際應用範例：**
```python
# Claude Desktop 透過 MCP 下指令
使用者: "往前走 1 公尺"
  ↓
ros-mcp-server: 發布 Twist 到 /cmd_vel
  ↓
go2_robot_sdk: 接收指令並驅動 Go2 硬體
```
