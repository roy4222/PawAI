**文件位置**：`docs/navigation/深度攝影機避障.md`  
**適用硬體**：Intel RealSense D435 + NVIDIA Jetson Orin Nano 8GB + Unitree Go2 Pro  
**建立日期**：2026-03-11  
**預計完成**：3/16 攻守交換前 Phase 0 上線  

---

## 1. 設計目標

### 1.1 核心定位

> **「導航避障是輔助能力，不搶主線互動控制權」**

本模組定位為 P2 加分項，優先確保不影響 P0 人臉/語音/AI 大腦的穩定性。

### 1.2 一句話目標

使用 **Intel RealSense D435** 深度攝影機，在 Jetson Orin Nano 8GB 上實現基礎近場避障能力，讓 Go2 能安全地在室內環境低速移動、繞行低矮障礙物。

### 1.3 三階段交付

| 階段 | 時間 | 目標 | 驗收標準 |
|------|------|------|----------|
| **Phase 0** | 現在→3/16 | 反應式避障 | 0.2-1.5m 障礙觸發停車，30 次測試 0 碰撞 |
| **Phase 1** | 3/17→4/6 | Nav2 局部導航 | 短距導航成功率 ≥ 90%，低矮障礙繞行 |
| **Phase 2** | 4/13 前 | nvblox + 語意 | 3D 地圖展示，語意分類（加分項） |

### 2.2 資料流架構

```
Phase 0: 反應式避障（立即實作）
═══════════════════════════════════════════════════════════════

[D435 深度相機]
    ↓ /camera/camera/depth/color/points (PointCloud2)
[depth_obstacle_filter_node] ← 新增
    - 深度濾波 (spatial/temporal/decimation)
    - ROI 裁切 (前向 3m)
    - 地面去除 (高度過濾)
    ↓ /camera/obstacle/points (過濾後 PointCloud2)
[nav2_collision_monitor] ← 新增
    - Stop Zone: 0.2-0.6m (立即停車)
    - Slow Zone: 0.6-1.5m (減速 30%)
    - 監測點雲侵入 polygon
    ↓ /cmd_vel_safe (Twist)
[twist_mux]
    - 優先級: /cmd_vel_safe > /cmd_vel_teleop > /cmd_vel_nav
    ↓ /cmd_vel
[go2_driver_node]
    ↓ 機器人運動

Phase 1: Nav2 局部導航（3/16 後）
═══════════════════════════════════════════════════════════════

[D435] → [depth_obstacle_filter_node]
    ↓ /camera/obstacle/points
[Nav2 Local Costmap]
    - VoxelLayer 接收 D435 PointCloud2
    - 與 LiDAR /scan 並行
    - 僅標記 0.05-1.2m 高度障礙
    ↓ /local_costmap/costmap
[controller_server]
    - DWB Local Planner
    - 低速設定 (max 0.25 m/s)
    ↓ /cmd_vel_nav
[twist_mux]
    ↓ /cmd_vel
[go2_driver_node]

Phase 2: 語意增強（4/13 前加分項）
═══════════════════════════════════════════════════════════════

[nvblox_node]
    - GPU 加速 3D 重建
    - ESDF 輸出
    ↓ /nvblox_node/projected_map
[semantic_obstacle_node]
    - 接收 YOLO/物體偵測結果
    - 區分: 人 / 紙箱 / 可忽略
    ↓ /semantic/costmap_overlay
[Nav2] → 整合語意層（Shadow Mode）
```

## 3. ROS2 Topic 契約

### 3.1 輸入 Topic（本模組訂閱）

| Topic | Message Type | 來源 | 說明 |
|-------|--------------|------|------|
| `/camera/camera/depth/color/points` | `sensor_msgs/PointCloud2` | D435 驅動 | 原始深度點雲 |
| `/camera/camera/depth/camera_info` | `sensor_msgs/CameraInfo` | D435 驅動 | 相機參數 |
| `/tf` | `tf2_msgs/TFMessage` | robot_state_publisher | 座標轉換 |
| `/scan` | `sensor_msgs/LaserScan` | pointcloud_to_laserscan | LiDAR 備援 |
| `/odom` | `nav_msgs/Odometry` | go2_driver_node | 里程計 |

### 3.2 輸出 Topic（本模組發布）

| Topic | Message Type | 目標 | 說明 |
|-------|--------------|------|------|
| `/camera/obstacle/points` | `sensor_msgs/PointCloud2` | collision_monitor, Nav2 | 過濾後障礙點雲 |
| `/camera/obstacle/scan` | `sensor_msgs/LaserScan` | Nav2 obstacle_layer | 可選：2D 投影 |
| `/cmd_vel_safe` | `geometry_msgs/Twist` | twist_mux | 安全層速度指令 |
| `/collision_monitor_state` | `nav2_msgs/CollisionMonitorState` | 監控 | 碰撞監控狀態 |

### 3.3 控制仲裁（twist_mux）

## 4. Phase 0 詳細設計

### 4.1 新增節點：`depth_obstacle_filter_node`

**職責**：從 D435 原始點雲提取有效障礙物點雲

**處理流程**：
1. **接收** `/camera/camera/depth/color/points`
2. **座標轉換** → `base_link` 座標系
3. **濾波處理**：
   - 範圍限制：0.2m ~ 3.0m（D435 可靠範圍）
   - 高度限制：0.05m ~ 1.2m（Go2 機身高度）
   - 左右限制：-1.0m ~ 1.0m（前方扇形）
4. **地面去除**：RANSAC 平面擬合或簡單高度閾值
5. **時間濾波**：多幀穩定性驗證（減少雜訊）
6. **發布** `/camera/obstacle/points`

**預期性能**：
- 輸入：424x240 @ 15Hz
- 輸出：10-15Hz
- CPU：Jetson Orin Nano < 15%

### 4.2 碰撞監控配置

```yaml
# collision_monitor_params.yaml
collision_monitor:
  ros__parameters:
    enabled: true
    base_frame_id: "base_link"
    odom_frame_id: "odom"
    cmd_vel_in_topic: "cmd_vel_nav"      # Nav2 輸入
    cmd_vel_out_topic: "cmd_vel_safe"     # 安全輸出
    transform_tolerance: 0.2
    source_timeout: 0.5
    base_shift_correction: true
    
    polygons: ["PolygonStop", "PolygonSlow"]
    
    # 緊急停車區：0.2-0.6m
    PolygonStop:
      type: "polygon"
      points: "[[0.6, 0.4], [0.6, -0.4], [0.2, -0.4], [0.2, 0.4]]"
      action_type: "stop"
      min_points: 5
      visualize: true
      polygon_pub_topic: "polygon_stop"
      
    # 減速區：0.6-1.5m
    PolygonSlow:
      type: "polygon"
      points: "[[1.5, 0.8], [1.5, -0.8], [0.6, -0.8], [0.6, 0.8]]"
      action_type: "slowdown"
      slowdown_ratio: 0.3
      min_points: 10
      visualize: true
      polygon_pub_topic: "polygon_slowdown"
      
    observation_sources: ["pointcloud"]
    
    pointcloud:
      type: "pointcloud"
      topic: "/camera/obstacle/points"
      min_height: 0.05
      max_height: 1.2
```

## 6. Launch 文件設計

### 6.1 新增 Launch 檔案

```python
# go2_robot_sdk/launch/realsense_obstacle.launch.py
# D435 驅動 + 深度過濾 + 碰撞監控

from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument
from launch.substitutions import LaunchConfiguration
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        DeclareLaunchArgument('enable_realsense', default_value='true'),
        DeclareLaunchArgument('enable_collision_monitor', default_value='true'),
        
        # RealSense D435
        Node(
            package='realsense2_camera',
            executable='realsense2_camera_node',
            name='realsense2_camera',
            parameters=[{
                'depth_module.depth_profile': '424x240x15',
                'enable_depth': True,
                'enable_color': False,
                'pointcloud.enable': True,
                'spatial_filter.enable': True,
                'temporal_filter.enable': True,
                'clip_distance': 3.0
            }],
            condition=IfCondition(LaunchConfiguration('enable_realsense'))
        ),
        
        # 深度障礙過濾
        Node(
            package='go2_robot_sdk',
            executable='depth_obstacle_filter_node',
            name='depth_obstacle_filter',
            parameters=[{
                'min_range': 0.2,
                'max_range': 3.0,
                'min_height': 0.05,
                'max_height': 1.2,
                'ground_threshold': 0.1
            }],
            condition=IfCondition(LaunchConfiguration('enable_realsense'))
        ),
        
        # 碰撞監控
        Node(
            package='nav2_collision_monitor',
            executable='collision_monitor',
            name='collision_monitor',
            parameters=['config/collision_monitor_params.yaml'],
            condition=IfCondition(LaunchConfiguration('enable_collision_monitor'))
        )
    ])
```

## 7. D435 失效處理策略

### 7.1 感知品質監測

```python
# depth_obstacle_filter_node 內部監測
class PerceptionQualityMonitor:
    def __init__(self):
        self.valid_depth_ratio_threshold = 0.5
        self.timeout_threshold = 0.5  # seconds
        self.pitch_roll_threshold = 25.0  # degrees
        
    def check_quality(self, pointcloud):
        quality_score = {
            'valid_ratio': self.calculate_valid_ratio(pointcloud),
            'update_latency': self.get_update_latency(),
            'overexposed': self.check_overexposure(),
            'pose_stability': self.check_pose_stability()
        }
        
        if quality_score['valid_ratio'] < self.valid_depth_ratio_threshold:
            return 'DEGRADED'
        if quality_score['update_latency'] > self.timeout_threshold:
            return 'TIMEOUT'
        return 'HEALTHY'
```

### 7.2 失效降級流程

```
檢測到 D435 品質退化
    ↓
發布 /perception_quality = DEGRADED
    ↓
collision_monitor 縮小 Stop Zone（更保守）
    ↓
降低最大速度至 0.1 m/s
    ↓
持續監測 3 秒
    ↓
[恢復] → 恢復正常參數
[持續退化] → 完全停止，等待人工介入
```

### 7.3 常見失效場景

| 場景 | 現象 | 對策 |
|------|------|------|
| **玻璃門** | 深度穿透，無回波 | 視為未知區域，停止 |
| **陽光直射** | IR 干擾，深度雜訊 | 降低信任度，尋找陰影區 |
| **低紋理牆面** | 深度稀疏 | 時間濾波，延長決策時間 |
| **鏡面反射** | 深度異常 | 有效點比例檢查 |
| **機身晃動** | 點雲抖動 | IMU 補償 + 保守閾值 |

## 9. 風險與對策

| 風險 | 機率 | 影響 | 對策 |
|------|:----:|:----:|------|
| D435 USB 不穩 | 中 | 高 | 必須支援 enable_realsense:=false 啟動 |
| Jetson 8GB 不足 | 中 | 高 | 解析度 424x240、15Hz、不開 color |
| nvblox 壓力過大 | 低 | 中 | Phase 2 列為加分項，不做硬依賴 |
| TF 不準確 | 中 | 高 | 精確標定 camera_link → base_link |
| 玻璃/反光失效 | 高 | 中 | 保守策略：未知即危險 |
| 控制權衝突 | 低 | 高 | 統一 twist_mux 仲裁 |
