# Contract v2.1 變更提案：gesture enum OK → Fist

**提案日期**：2026-03-18
**提案者**：Agent 2（手勢辨識研究）
**狀態**：PENDING — 待 System Architect 核准
**影響範圍**：`/event/gesture_detected` schema

---

## 變更內容

`interaction_contract.md` §4.3 `/event/gesture_detected` 的 gesture enum：

```diff
- "gesture": { "type": "string", "enum": ["wave", "stop", "point", "ok"] }
+ "gesture": { "type": "string", "enum": ["wave", "stop", "point", "fist"] }
```

---

## 理由

### 1. OK 手勢可靠度不足（Tier 3）

2026-03-17 社群調查結果：

| 手勢 | Tier | 說明 |
|------|:----:|------|
| stop ✋ | 1 | 五指展開，特徵極度明顯 |
| point 👉 | 1 | 食指獨立伸展，特徵獨特 |
| **fist ✊** | **1** | 所有指尖靠近手掌，與任何張開手勢對比強烈 |
| **~~ok~~ 👌** | **3** | 拇指食指距離判定，DWPose 遮擋場景 AP 降到 ~45% |

OK 手勢依賴拇指尖與食指尖的精確距離，在以下場景容易誤判：
- DWPose/RTMPose 遮擋場景（hand AP 從 78% 降到 ~45%）
- 側面角度（拇指/食指重疊不可見）
- 與 Closed Fist 混淆（三指未完全展開時）

### 2. Fist 是業界公認最穩定手勢之一

- MediaPipe Gesture Recognizer 8 種內建手勢中 Closed_Fist 列為首位
- NVIDIA trt_pose_hand 6 類內建手勢中 fist 列為首位
- 分類邏輯極簡：所有 fingertip-to-MCP 距離 < 閾值

### 3. Demo B 成功率影響

目標：4 種手勢成功率 ≥ 70%。OK 作為 Tier 3 手勢是最大風險點。
替換為 Tier 1 的 Fist 後，4 種手勢全部是 Tier 1 或有成熟時序方案（wave），大幅降低 Demo B 失敗風險。

---

## 過渡期相容方案

核准前，實作層採用內部映射，不破壞既有 consumer：

```python
# gesture_classifier 內部用 "fist"
# 發布 /event/gesture_detected 時映射回 v2.0 enum
GESTURE_COMPAT_MAP = {"fist": "ok"}  # v2.0 相容

def publish_gesture(gesture: str):
    compat_gesture = GESTURE_COMPAT_MAP.get(gesture, gesture)
    # 發布 compat_gesture 到 /event/gesture_detected
```

核准後移除映射，直接發布 `"fist"`。
