## 🎯 C組 → A組: VLM 輸出格式

### JSON Schema (標準約定)

```json
{
  "detected": boolean,           // 是否偵測到目標物
  "object": string,              // 物體類別名稱 (COCO 80類)
  "bbox": [x, y, width, height], // Bounding Box (左上角+寬高)
  "center": [u, v],              // 框框中心點像素座標
  "confidence": float            // 信心度 (0.0 ~ 1.0)
}
```

### 座標系統定義 (Critical!)

**影像座標系 (Image Coordinate System)**

```
(0,0) ─────────────────► u (width)
  │
  │    +--------+
  │    |  物體  |  ← bbox: [x, y, w, h]
  │    |   ●    |  ← center: [u, v]
  │    +--------+
  │
  ▼ v (height)
```

**關鍵參數:**
- **原點 (Origin):** 影像左上角 `(0, 0)`
- **u軸 (橫向):** 向右為正 (範圍: `0` → `image_width`)
- **v軸 (縱向):** 向下為正 (範圍: `0` → `image_height`)
- **bbox 格式:** `[x, y, width, height]` (⚠️ **非** `[x, y, x2, y2]`)
- **center 計算公式:** `[x + width/2, y + height/2]`

### 範例 1: 偵測到杯子

**輸入影像:** 640x480 解析度

**VLM 輸出:**
```json
{
  "detected": true,
  "object": "cup",
  "bbox": [200, 150, 80, 120],
  "center": [240, 210],
  "confidence": 0.92
}
```

**驗證計算:**
- `center[0] = 200 + 80/2 = 240` ✅
- `center[1] = 150 + 120/2 = 210` ✅

### 範例 2: 未偵測到目標

**VLM 輸出:**
```json
{
  "detected": false,
  "object": null,
  "bbox": null,
  "center": null,
  "confidence": 0.0
}
```

### 範例 3: 多物體偵測 (返回陣列)

若場景中有多個物體,C組應回傳陣列:

```json
{
  "detected": true,
  "objects": [
    {
      "object": "cup",
      "bbox": [200, 150, 80, 120],
      "center": [240, 210],
      "confidence": 0.92
    },
    {
      "object": "bottle",
      "bbox": [400, 200, 60, 150],
      "center": [430, 275],
      "confidence": 0.88
    }
  ]
}
```

**A組處理邏輯:**
- 若指定尋找 "cup",則只取第一個物件座標
- 若未指定,則優先取 confidence 最高者

## 📡 C組 → B組: VLM API 規範

### HTTP API 端點

**Base URL:** `http://<GPU_SERVER_IP>:8000`

#### 1. POST /detect

**功能:** 上傳影像,回傳物體偵測結果

**Request:**
```http
POST /detect HTTP/1.1
Content-Type: multipart/form-data

image: <binary_image_data>
target_name: "cup" (optional)
```

**Response (成功):**
```json
{
  "status": "success",
  "result": {
    "detected": true,
    "object": "cup",
    "bbox": [200, 150, 80, 120],
    "center": [240, 210],
    "confidence": 0.92
  },
  "inference_time": 0.023  // 單位: 秒
}
```

**Response (失敗):**
```json
{
  "status": "error",
  "message": "Model inference failed",
  "error_code": "VLM_001"
}
```

#### 2. GET /health

**功能:** 健康檢查

**Response:**
```json
{
  "status": "healthy",
  "model_loaded": true,
  "gpu_available": true,
  "vram_usage": "12.5 GB / 48.0 GB"
}
```

### Python 呼叫範例 (A組使用)

```python
import requests

# 讀取影像
with open('test.jpg', 'rb') as f:
    image_data = f.read()

# 發送請求
response = requests.post(
    'http://192.168.1.100:8000/detect',
    files={'image': image_data},
    data={'target_name': 'cup'}
)

# 解析結果
result = response.json()
if result['status'] == 'success' and result['result']['detected']:
    center = result['result']['center']
    print(f"物體中心座標: {center}")
```

## 🔄 A組: 座標轉換流程

### 輸入 (from C組)

```python
{
  "center": [u, v]  // 像素座標 (影像座標系)
}
```

### 處理流程

**Phase 2 實作: Level 1 地面假設法**

```python
# 1. 訂閱 camera_info 取得內參矩陣 K
K = camera_info.K  # 3x3 matrix

# 2. 計算歸一化座標 (Normalized Coordinates)
u_normalized = (u - K[0,2]) / K[0,0]
v_normalized = (v - K[1,2]) / K[1,1]

# 3. 假設物體在地面 (z = 0 in map frame)
# 使用 tf2 取得相機高度 h 與俯仰角 θ
camera_height = h
camera_pitch = θ

# 4. 計算射線與地面交點
distance = camera_height / tan(θ + atan(v_normalized))
x_world = distance * cos(yaw + atan(u_normalized))
y_world = distance * sin(yaw + atan(u_normalized))
z_world = 0.0

# 5. 發佈世界座標
pose = PoseStamped()
pose.header.frame_id = "map"
pose.pose.position.x = x_world
pose.pose.position.y = y_world
pose.pose.position.z = z_world
```

### 輸出 (to Nav2)

```python
# ROS2 Topic: /target_pose
# Message Type: geometry_msgs/PoseStamped

{
  "header": {
    "frame_id": "map",
    "stamp": <current_time>
  },
  "pose": {
    "position": {"x": 2.5, "y": 1.0, "z": 0.0},
    "orientation": {"x": 0, "y": 0, "z": 0, "w": 1}
  }
}
```

## 🖥️ B組: 資源管理介面

### GPU 資源切換腳本

**目的:** RTX 8000 (48GB) 可同時運行 Isaac Sim + VLM,但為穩定性提供切換機制

#### 1. start_sim.sh

```bash
#!/bin/bash
# 啟動 Isaac Sim,停止 VLM API

echo "停止 VLM API..."
pkill -f inference_server.py

echo "啟動 Isaac Sim..."
cd ~/go2_omniverse
./isaac_sim.sh --headless --streaming native

echo "Isaac Sim 已啟動,VNC 連線可用"
```

#### 2. start_vlm.sh

```bash
#!/bin/bash
# 啟動 VLM API,停止 Isaac Sim

echo "停止 Isaac Sim..."
pkill -f isaac_sim

echo "啟動 VLM API..."
cd ~/vlm_server
source venv/bin/activate
uvicorn inference_server:app --host 0.0.0.0 --port 8000

echo "VLM API 已啟動於 http://0.0.0.0:8000"
```

### 監控介面

```bash
# 即時監控 GPU 狀態
watch -n 1 nvidia-smi

# 或使用更友善的 nvtop
nvtop
```

**VRAM 使用預估:**
- Isaac Sim: 15-20 GB
- VLM (YOLO): 10-15 GB
- 系統保留: 5 GB
- **總計:** 30-40 GB (在 48 GB 限制內安全)

## 🔒 介面變更流程

### 變更等級

| 等級 | 影響範圍 | 審批流程 | 範例 |
|------|---------|---------|------|
| 🔴 Breaking | 破壞現有功能 | **週會全組同意** | 改變座標原點定義 |
| 🟡 Non-Breaking | 新增功能 | 發起組通知即可 | 新增 `timestamp` 欄位 |
| 🟢 內部優化 | 不影響介面 | 無需通知 | 優化推論速度 |

### 變更提案範本

```markdown
## 介面變更提案

**提案組別:** C組
**提案日期:** 2025/11/XX
**變更等級:** 🟡 Non-Breaking

**變更內容:**
在 VLM 輸出 JSON 中新增 `timestamp` 欄位

**變更理由:**
方便 A 組進行數據同步與除錯

**變更前:**
```json
{
  "detected": true,
  "object": "cup"
}
```

**變更後:**
```json
{
  "detected": true,
  "object": "cup",
  "timestamp": "2025-11-23T10:30:45.123Z"
}
```

**影響評估:**
- A組: 可選用,不影響現有邏輯
- B組: 無影響
```

## 🧪 測試案例

### 案例 1: 正常偵測流程

1. **C組 VLM** 收到 640x480 影像
2. **偵測結果:**
   ```json
   {
     "detected": true,
     "object": "cup",
     "center": [320, 240],
     "confidence": 0.95
   }
   ```
3. **A組接收** center [320, 240]
4. **座標轉換** → world [2.5, 0.0, 0.0]
5. **Nav2 導航** → 機器狗走到杯子前

**預期結果:** ✅ 成功找到物體

### 案例 2: 未偵測到物體

1. **C組 VLM** 收到影像,但無目標物
2. **偵測結果:**
   ```json
   {
     "detected": false
   }
   ```
3. **A組處理** → 繼續巡邏,不進行座標轉換

**預期結果:** ✅ 系統正常運作,不會 Crash

### 案例 3: 座標邊界測試

**輸入:** 物體在影像右下角

```json
{
  "center": [630, 470]  // 接近 640x480 邊界
}
```

**A組驗證:**
- `0 <= u < 640` ✅
- `0 <= v < 480` ✅
- 若超出範圍 → 記錄 Warning,拒絕轉換
