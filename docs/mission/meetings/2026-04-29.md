- **原問題**：雷達原本用魔鬼氈黏在 Go2 背上，行走時搖晃導致 SLAM 地圖重複疊加，地圖會越掃越大

## 四、LLM 架構調整（重要決策）

### 從本地模型轉向雲端 API

- **原方案**：GPU server 跑 Qwen 2.5 7B 為主力
- **問題**：
  - Qwen 2.5 7B 智商不夠，容易混淆「聊天 vs 動作指令」
  - 上下文太短，沒有記憶力
  - GPU server 跑大模型不穩定，常斷線
  - Qwen 3.5 有思考過程，速度慢且會爆
- **新方案**：用 OpenRouter API 為主力
  - 燭光計畫的設備使用費可拿來買 API credits（秘書已確認可行）
  - 候選模型：Gemini 2.5 Flash、DeepSeek V4（0.14 美元等級超便宜）、Kimi K2 等
  - GPU server 降為 fallback

### 斷網應對策略

- 真的全斷網時，反正本地 GPU 也會掛（現在還是雲端 GPU）
- 純本機（Jetson）能跑的只有：手勢、姿勢、人臉、物體辨識
- 全斷網展示策略：用假 LLM 模板，維持機器狗能執行固定動作

### TTS 也想升級

- 現在 Edge TTS 聽起來跟 Google 小姐差不多
