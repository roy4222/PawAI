## 一、現況簡報（Roy 主講）

### 1. 硬體配置

- **本體**：Unitree Go2 Pro
- **運算單元**：NVIDIA Jetson Orin Nano Super（背載）
- **感測器**：前方 Intel RealSense 相機、頂部新裝 LiDAR
- **音訊**：外接喇叭（取代 Go2 不穩定的內建喇叭）

### 2. 系統架構（三層式）

- **第一層 (Top)**：互動意圖判斷 — 決定該說什麼話、做什麼動作
- **第二層 (Middle)**：Face / Vision (姿勢、手勢、物件辨識) / Speech / Memory
- **第三層 (Bottom / Safety)**：動作安全層 — **LLM 不可直接操控移動**，需經安全層審核，避免危險動作

### 3. 模型分工

- **Cloud（GPU 伺服器）**：Qwen 2.5 7B (LLM)、SenseVoice + Whisper (ASR)、TTS
- **Local（Jetson）**：Face Recognition (InsightFace)、Pose / Gesture (MediaPipe)、Object (YOLOv26n)、STT

### 4. Dashboard 現場展示（黃旭製作的 Web Dashboard）

- 即時看到 Go2 視角畫面、六大功能模組狀態（face / voice / gesture / pose / object / speech）
- 現場展示功能：
  - 人臉辨識打招呼（認得 Roy 並叫名字）
  - 手勢辨識（比讚 → happy 反應）
  - 物件辨識（cup / phone / book 等）
  - 姿勢辨識（站立 / 蹲下）
  - 語音對話（問天氣、講笑話、自我介紹等）

### 5. 目前優先順序

- **主打**：人機互動（打招呼、聊天、reaction）— 約佔 70%
- **次要（後期再做）**：守護功能（開門、辨陌生人、跟隨巡邏）— 約佔 30%，需等 LiDAR 與導航成熟

### 6. 已知問題

- LiDAR 裝在頂部需處理穩定性與固定問題（目前用魔鬼氈，不夠穩），晃動會造成定位誤差
- 內建喇叭不穩，已改用外接喇叭
- Go2 連網會自動更新韌體，曾造成 debug 失敗（已列為操作風險）

## 二、Annie 教授提問與建議

### 1. 關於對話設計

**Q**：Action 與 Conversation 如何區分？如何避免 LLM 誤判把聊天當成移動指令？

**A (Roy)**：目前架構刻意分層，純對話走 LLM，動作走 rule-based + safety layer。
LLM 不可直接觸發移動，避免「誤解語意 → 亂跑」的風險。

### 2. 關於對話自然度

- Annie 觀察：目前比較像「一問一答」，缺少 ChatGPT 式的延續性互動（主動反問、引導下一步）
- 建議方向：讓對話更像「人 vs 寵物狗」的情感互動，而非純粹指令式問答

### 3. 關於情感運算 (Affective Computing) ⭐

Annie 介紹同校老師研究的 **Affect Computing** 領域：

- 使用 audio-visual dataset 偵測使用者情緒（臉部表情 + 聲音語氣）
- 機器狗作為居家陪伴裝置，**情緒價值**很重要
- 建議未來可加入：
  - 從語氣判斷使用者今天 sad / happy / upset
  - 機器狗主動給予 emotional support 回應
    - 例：「你今天看起來不太開心，要不要出去走走？」
  - 文字 → 語音轉換時保留情緒 metadata（目前轉文字後情緒資訊就遺失）

### 4. 關於 TTS 升級建議

- Roy 提到可以考慮換用 **Gemini 2.5 Flash TTS**（現場展示官方機器人 demo）
- 語氣自然度顯著高於目前的 TTS，且支援多語言切換
- 待議：Go2 頭部與身體連動，搖頭等細微動作受限，但語氣本身可大幅提升互動感

### 5. Annie 分享其學生作品

- 一個 Web 介面的虛擬 Avatar（人形頭像），用於對學生提供心理壓力支援的對話
- 用來對照「有實體機器狗 vs 純虛擬 avatar」兩種陪伴模式的差異
