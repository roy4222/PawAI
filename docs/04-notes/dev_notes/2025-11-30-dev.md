```

    <ParticipantIndex>auto</ParticipantIndex>
    <MaxAutoParticipantIndex>120</MaxAutoParticipantIndex>
    <SPDPInterval>1000ms</SPDPInterval>
</Discovery>
```

**驗證結果：**
- ✅ `robot.launch.py` 成功啟動所有節點

## 📊 技術收穫

### 1. CycloneDDS MaxAutoParticipantIndex 參數

**關鍵發現：**
- `MaxAutoParticipantIndex` 決定了同時運行的 ROS2 節點數量上限
- 預設值通常只有 10-20，對於複雜系統（Nav2 + SLAM）不夠
- 建議設定為 120 以上

**計算公式：**
```
所需數量 = 基礎節點 + SLAM 節點 + Nav2 節點 + 自定義節點 + 緩衝
         = 5 + 10 + 15 + 5 + 85 (緩衝) = 120
```

---

### 2. CycloneDDS 版本兼容性

**兼容性對照表：**

| XML 標籤 | 舊版支援 | 新版支援 | 建議 |
|---------|---------|---------|------|
| `<NetworkInterfaceAddress>IP</NetworkInterfaceAddress>` | ✅ | ✅ | 使用 (兼容性好) |
| `<NetworkInterface>eth0</NetworkInterface>` | ❌ | ✅ | 避免 (舊版不支援) |
| `<AllowMulticast>true</AllowMulticast>` | ❌ | ✅ | 避免 (舊版不支援) |

**教訓：** 跨平台開發時，使用最保守的 XML 標籤以確保兼容性

---

### 3. Zsh vs Bash 的跨 Shell 陷阱

**問題發現：**
- 在 Mac VM 設定 ROS2 環境變數時，使用 `source setup.bash` 在 zsh 環境下導致路徑解析錯誤
- 症狀：某些環境變數未正確載入，導致 `ros2` 指令找不到套件

**解決方案：**
```bash
# 錯誤做法 (在 zsh 環境)
source install/setup.bash

# 正確做法 (在 zsh 環境)
source install/setup.zsh
```

**教訓：**
- 不同 Shell 對環境變數與路徑的處理方式不同
- ROS2 提供了多種 Shell 的 setup 腳本 (`setup.bash`, `setup.zsh`, `setup.sh`)
- **最佳實踐**：檢查當前 Shell (`echo $SHELL`)，使用對應的 setup 腳本

---

### 4. UTM 虛擬機網路架構

**UTM 網路模式對比：**

| 模式 | 優點 | 缺點 |
|------|------|------|
| Shared Network (NAT) | 簡單，VM 自動上網 | VM 與外部設備不在同網段 |
| Bridged (橋接) | VM 直接取得實體 IP | 需手動配置 IP |
| Host-only | 安全，僅 Host ↔ VM | VM 無法上網 |

**當前問題：** 使用 NAT 模式，VM 與機器狗不在同網段

**解決方案（待實施）：**
1. Plan A：改用 Bridged Mode
2. Plan B：使用 rosbag 錄製數據
3. Plan C：在 Mac 主機上直接運行 ROS2
