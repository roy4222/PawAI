
8. **嘗試 6：查閱 CycloneDDS 官方文件**
   - 發現 `NetworkInterfaceAddress` 已被棄用
   - 新版語法應使用 `<Interfaces><NetworkInterface name="..."/></Interfaces>`
   - 參考文件：
     - [Eclipse CycloneDDS Issue #1434](https://github.com/eclipse-cyclonedds/cyclonedds/issues/1434)
     - [Configuration File Reference](https://cyclonedds.io/docs/cyclonedds/latest/config/config_file_reference.html)

9. **最終解決方案：使用新版 CycloneDDS 語法 + 強制 Loopback**
   - 建立 `/home/roy422/local_only_v2.xml`：
     ```xml
     <?xml version="1.0" encoding="UTF-8" ?>
     <CycloneDDS xmlns="https://cdds.io/config">
         <Domain>
             <General>
                 <Interfaces>
                     <NetworkInterface name="lo"/>
                 </Interfaces>
                 <AllowMulticast>false</AllowMulticast>
             </General>
             <Discovery>
                 <ParticipantIndex>auto</ParticipantIndex>
                 <MaxAutoParticipantIndex>120</MaxAutoParticipantIndex>
                 <Peers>
                     <Peer address="127.0.0.1"/>
                 </Peers>
             </Discovery>
         </Domain>
     </CycloneDDS>
     ```

     ```bash

     export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
     export CYCLONEDDS_URI=/home/roy422/local_only_v2.xml
     ros2 run demo_nodes_cpp talker

     # Terminal 2
     export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
     export CYCLONEDDS_URI=/home/roy422/local_only_v2.xml
     ros2 run demo_nodes_cpp listener
     ```
   - **結果：✅ 成功！** Listener 收到 "Hello World" 訊息

   - **驗證步驟 2：Go2 驅動測試**
     ```bash

     ```
   - **結果：✅ 完全成功！**
     - 看到所有 ROS2 節點（`go2_driver_node`、`robot_state_publisher` 等）
     - 看到所有感測器 topics（`/scan`、`/point_cloud2`、`/camera/image_raw` 等）

### 🎯 問題根源總結

**核心問題：雙網卡環境下的 DDS 路由選擇混亂**

1. **現象：**
   - `ros2 node list` 可以看到節點（Discovery 成功）
   - `ros2 topic hz` 收不到數據（Data Transfer 失敗）

2. **根本原因：**
   - VM 有 3 個網卡（`lo`、`enp0s1`、`enp0s2`）
   - CycloneDDS 沒有明確指定網卡時，會自動選擇
   - **Terminal 1（驅動）可能選擇 enp0s1 發送數據**
   - **Terminal 2（監控）可能選擇 enp0s2 監聽數據**
   - 結果：Discovery 封包透過 multicast 廣播成功，但 Data Transfer 走錯網卡

3. **為什麼 Discovery 成功但 Data Transfer 失敗？**
   - **Discovery（節點發現）：** 使用 UDP multicast，廣播到所有網卡
   - **Data Transfer（數據傳輸）：** 使用 unicast，必須指定正確的 IP 和網卡

4. **解決方案：**
   - 強制所有 ROS2 通訊走 `lo`（127.0.0.1）
   - 使用新版 CycloneDDS 語法 `<NetworkInterface name="lo"/>`
   - 確保所有 Terminal 使用**相同的 XML 配置檔案**

### 📊 技術收穫

#### 1. CycloneDDS 配置最佳實踐

**舊版語法（已棄用）：**
```xml
<General>
    <NetworkInterfaceAddress>127.0.0.1</NetworkInterfaceAddress>
</General>
```

**新版語法（推薦）：**
```xml
<General>
    <Interfaces>
        <NetworkInterface name="lo"/>
    </Interfaces>
</General>
```

**多網卡指定方式：**
```xml
<Interfaces>
    <NetworkInterface name="eth0" priority="10"/>
    <NetworkInterface name="wlan0" priority="5"/>
</Interfaces>
```

#### 2. ROS2 DDS 除錯方法

**檢查環境變數：**
```bash
echo $RMW_IMPLEMENTATION      # 應為 rmw_cyclonedds_cpp
echo $CYCLONEDDS_URI          # 應指向正確的 XML 檔案
```

**檢查網卡狀態：**
```bash
ip addr show                  # 列出所有網卡和 IP
ip route show                 # 列出路由表
```

**測試 DDS 通訊：**
```bash
# 使用 ROS2 內建的 talker/listener
ros2 run demo_nodes_cpp talker
ros2 run demo_nodes_cpp listener
```

**檢查 ROS2 節點和 topics：**
```bash
ros2 node list                # 列出所有節點（測試 Discovery）
ros2 topic list               # 列出所有 topics
ros2 topic hz /topic_name     # 測試數據傳輸
ros2 topic echo /topic_name   # 查看實際數據內容
```

#### 3. 雙網卡環境的陷阱

**陷阱 1：DHCP 自動分配 IP**
- Go2 機器狗的 DHCP 會自動分配 IP 給連接的設備
- 即使手動設定靜態 IP，DHCP 仍可能分配 secondary IP
- 導致同一張網卡有多個 IP，CycloneDDS 可能混亂

**陷阱 2：預設路由選擇**
- Linux 的預設路由會影響 DDS 封包走向
- 即使 XML 指定了網卡，路由表仍可能導致封包走錯路徑

**陷阱 3：環境變數未同步**
- 不同 Terminal 如果使用不同的 `CYCLONEDDS_URI`，會在不同的 DDS Domain
- 必須確保所有 Terminal 使用**完全相同**的環境變數

**最佳解決方案：**
- 在單機開發時，**一律使用 loopback (`lo`)**
- 只有在跨機器通訊時，才指定實體網卡
- 使用明確的 XML 配置，不依賴自動偵測
