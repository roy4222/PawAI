## 📅 今日重大發現：Go2 Pro 有線模式的真相

經過深入的技術分析與外部資料驗證，我們確認了關於 Go2 Pro 硬體與連線的關鍵限制，這將導向專案開發策略的重大調整。

### 🚨 關鍵結論：Go2 Pro 有線連接無法傳輸感測數據

1. **硬體缺失：** Go2 Pro 版本內部**沒有內建 Jetson Orin 運算模組**（僅 EDU 版本配備）。
2. **通訊限制：** 
   - 有線網卡 (192.168.123.161) 僅連接到底層 MCU，用於控制指令發布。
   - 所有感測器資料（LiDAR、IMU、Camera）皆由 MCU 處理，且僅透過 WiFi WebRTC 協議傳輸。
   - CycloneDDS 模式完全是為 EDU 版本設計的，Pro 版本在有線網段下不會發布感測器 Topics。
3. **驅動現狀：** `go2_ros2_sdk` 的 CycloneDDS 支援僅為框架，內部的 Callbacks (LiDAR, Pose, LowState) 皆為空實作 (`pass`)。

### 📊 Go2 Pro vs EDU 核心差異

| 功能 | Go2 Pro | Go2 EDU |
|------|---------|---------|
| 有線網卡 | ✅ 有 | ✅ 有 |
| 有線 ROS2 Topics | ❌ 無 (無 Jetson) | ✅ 有 |
| WiFi WebRTC | ✅ 有 | ✅ 有 |
| 內建運算 | ❌ 無 Jetson | ✅ Jetson Orin Nano / NX |
| 單價差距 | 約 $2,800 USD | >$16,000 USD |

## ⚙️ 技術策略調整：從「純有線」轉向「雙通道 Hybrid」

基於硬體限制，我們修正了原本「全面轉有線」的計畫，改為更務實的兩階段方案。

### 🚨 文檔與現實的矛盾 (Gap Analysis)
*   **預期**：CycloneDDS 有線可解決所有傳輸問題（延遲 <1ms）。
*   **現實**：Go2 Pro 有線僅連到 MCU，**無法傳輸相機影像**。意味著「純有線」會讓機器人失明，無法執行尋物任務。

### 🚀 兩階段實施路徑

#### 第一階段：WiFi Client 模式（Plan B - 保底方案）
- **核心目標**：確保「看得見」且不掉線。
- **優化手段**：
    - **網路拓樸**：Go2 (WiFi) 連接高品質 WiFi 6 路由器，Mac 則透過有線連接路由器。
    - **流量抽幀**：在驅動層將相機影像從 30fps 降至 10fps（減少 >60% 頻寬壓力），LiDAR 維持 7Hz 保證建圖精度。

#### 第二階段：Hybrid Mode 混合模式（Plan A - 進階方案）
- **核心目標**：實現「看得見環境 + 動得精準」。
- **實現方式**：
    - **視覺數據流**：WebRTC 走 WiFi（感知層）。
    - **運動指令流**：CycloneDDS 走有線（控制層，延遲 <1ms）。
- **當前進度**：已建立 `scripts/test_wired_control.py` 驗證腳本，準備測試 Pro 版本有線介面是否接受運動指令。
