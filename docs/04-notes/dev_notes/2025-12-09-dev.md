#### 1. 建立 `start_mcp.sh` (15 分鐘)

**位置：** `~/ros2_ws/src/elder_and_dog/start_mcp.sh`

```bash
#!/bin/zsh
# 一鍵啟動 MCP 所需的所有服務

set -e
SCRIPT_DIR=$(cd "$(dirname "$0")" && pwd)

echo "🐕 啟動 Go2 MCP 控制系統..."

# 檢查 tmux session 是否已存在
if tmux has-session -t go2_mcp 2>/dev/null; then
    echo "⚠️  Session 'go2_mcp' 已存在！"
    echo "📋 選項："
    echo "  1. tmux attach -t go2_mcp  （連接到現有 session）"
    echo "  2. tmux kill-session -t go2_mcp  （刪除後重新啟動）"
    exit 1
fi

# 使用 tmux 管理多個 Terminal
tmux new-session -d -s go2_mcp

# Pane 0: rosbridge (啟動快，約 2-3 秒)
tmux send-keys -t go2_mcp:0 "cd $SCRIPT_DIR && zsh phase1_test.sh bridge" C-m

# 水平分割 - Pane 1: Driver (WebRTC 握手需要時間)
tmux split-window -h -t go2_mcp:0
tmux send-keys -t go2_mcp:0.1 "sleep 3 && cd $SCRIPT_DIR && zsh start_go2_simple.sh" C-m

# 垂直分割 - Pane 2: snapshot_service (等 Driver 就緒後啟動)
tmux split-window -v -t go2_mcp:0.0
tmux send-keys -t go2_mcp:0.2 "sleep 8 && source ~/ros2_ws/install/setup.zsh && export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp && export CYCLONEDDS_URI=/home/roy422/local_only_v2.xml && ros2 run go2_robot_sdk snapshot_service" C-m

# 連接到 session
echo "✅ 所有服務已啟動中..."
echo "⏳ 請等待約 12 秒讓系統就緒..."
echo "📋 連接 tmux session: tmux attach -t go2_mcp"
echo "🛑 停止服務: tmux kill-session -t go2_mcp"

tmux attach -t go2_mcp
```

**使用方式：**
```bash
chmod +x start_mcp.sh
zsh start_mcp.sh

# 停止服務
tmux kill-session -t go2_mcp
```

```markdown
# Go2 機器狗控制 Prompt

你是 Go2 機器狗的控制 AI，可以透過 MCP 工具控制機器狗。

## 系統確認（每次對話開始時必須執行）

### 1. 確認連線狀態
使用 `get_topics()` 檢查系統：
- 確認至少有 **20+ topics**
- 確認存在：`/cmd_vel`, `/camera/image_raw`, `/capture_snapshot`

### 2. 如果連線成功
告訴使用者：「系統已就緒，機器狗已連線。」
使用 `call_service('/capture_snapshot', 'std_srvs/Trigger')` 拍照
顯示影像分析結果

### 3. 如果連線失敗
告訴使用者：「⚠️ 系統未就緒，請先執行 `zsh start_mcp.sh`」
**不要嘗試控制機器狗**

## 移動控制
- 前進：`publish_once('/cmd_vel', 'geometry_msgs/Twist', {"linear": {"x": 0.2}})`
- 左轉：`publish_once('/cmd_vel', 'geometry_msgs/Twist', {"angular": {"z": 0.3}})`
- 右轉：`publish_once('/cmd_vel', 'geometry_msgs/Twist', {"angular": {"z": -0.3}})`
- 停止：`publish_once('/cmd_vel', 'geometry_msgs/Twist', {"linear": {"x": 0.0}})`

## 安全限制
- linear.x: 0.1 ~ 0.3 m/s（最大速度）
- angular.z: 0.3 ~ 0.5 rad/s（最大旋轉）
- **移動前必須先拍照確認環境**

## 避障策略
1. 拍照分析環境
2. 如果前方有障礙物 → 告知使用者並詢問是否繞行
3. 左轉或右轉避開
4. 再次拍照確認
5. 前進

## 精確移動（使用 odometry）
1. 記錄起始位置：`subscribe_once('/odom', 'nav_msgs/Odometry')`
2. 開始移動
3. 定期檢查當前位置
4. 計算移動距離 = sqrt((x2-x1)² + (y2-y1)²)
5. 達到目標距離時停止
```

## ⚠️ 已解決問題

### DDS Discovery 失敗（已解決 ✅）

**現象：**
- `ros2 topic list` / `ros2 service list` 無限卡住
- 所有節點都在運行（rosbridge、go2_driver、snapshot_service）
- 但節點之間無法互相發現

**嘗試過的解法（全部失敗）：**
1. ❌ 改用 `cyclonedds_dual.xml`（AllowMulticast=true）
2. ❌ 修正網路路由（預設閘道改到 192.168.1.1）
3. ❌ 重啟機器狗
4. ❌ 註解 `~/.zshrc` 的 CycloneDDS 設定
5. ❌ 在腳本加 `unset RMW_IMPLEMENTATION && unset CYCLONEDDS_URI`

**🎯 真正的根因：**
```
ros2 daemon 會 cache DDS 設定！
即使修改了 ~/.zshrc 或腳本，daemon 還是用舊的 CycloneDDS。
```

查看 daemon 進程：
```bash
ps aux | grep ros2-daemon
# 輸出：ros2-daemon --rmw-implementation rmw_cyclonedds_cpp  ← 這就是問題！
```

**✅ 最終解法：**
```bash
# 1. 殺掉舊的 ros2 daemon（這是關鍵！）
pkill -9 -f "ros2-daemon"

# 2. 清除環境變數
unset RMW_IMPLEMENTATION
unset CYCLONEDDS_URI

# 3. 測試（應該 2-3 秒內返回）
ros2 topic list  # ✅ 20+ topics
ros2 service list | grep snapshot  # ✅ /capture_snapshot
```

**修改的檔案：**
- `~/.zshrc`：註解掉 CycloneDDS 設定（避免未來再發生）
- `start_mcp.sh`：加入 `unset` 指令
- `start_go2_simple.sh`：加入 `unset` 指令
- `phase1_test.sh`：`load_ros_env()` 改用 FastDDS

## 📋 W7 Day 2 計畫

### 🎯 核心任務（預估 4-6 小時）

#### 1. 測試實際移動控制（1 小時）
- **目標：** `/cmd_vel` 前進/後退/左轉/右轉/停止全測試
- **風險：** Wi-Fi 延遲導致指令丟失（機率：中）
- **驗收標準：** 機器狗能精確執行 5 個基礎動作
- **測試指令：**
  ```
  publish_once('/cmd_vel', 'geometry_msgs/Twist', {"linear": {"x": 0.2}})  # 前進
  publish_once('/cmd_vel', 'geometry_msgs/Twist', {"angular": {"z": 0.3}})  # 左轉
  publish_once('/cmd_vel', 'geometry_msgs/Twist', {"linear": {"x": 0}, "angular": {"z": 0}})  # 停止
  ```

#### 2. 測試 Odometry 精確移動（2 小時）
- **目標：** 前進 1m，誤差 < 15cm
- **風險：** 地面濕滑導致打滑（機率：低）
- **驗收標準：** 10 次測試，8 次誤差 < 15cm
- **方法：**
  1. subscribe_once('/odom') 記錄起點
  2. 發送 /cmd_vel 前進
  3. 每 2 秒 subscribe_once('/odom') 檢查距離
  4. 達到目標距離時發送停止指令

#### 3. 完整避障流程測試（1-3 小時）
- **情境 1：** 正前方障礙物 → 右轉避開 → 再拍照確認 → 前進
- **情境 2：** 左側障礙物 → 直行（保持安全距離）
- **情境 3：** 前方無路（死胡同）→ 告知使用者並等待指令
- **驗收標準：** 3 個情境各成功 2 次
