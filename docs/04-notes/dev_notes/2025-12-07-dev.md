## 📋 Phase 2.1：snapshot_service 開發

### 目標
讓 LLM 能透過 MCP 擷取相機影像，回傳 base64 JPEG。

### 規格
```python
# 預期 API
# MCP 呼叫: call_service('/capture_snapshot', 'std_srvs/Trigger')
# 回傳: {'success': True, 'message': '<base64 encoded JPEG>'}
```

### 檔案位置
```
src/go2_robot_sdk/go2_robot_sdk/snapshot_service.py
```

### 基本架構
```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from std_srvs.srv import Trigger
from cv_bridge import CvBridge
import cv2
import base64

class SnapshotService(Node):
    def __init__(self):
        super().__init__('snapshot_service')
        self.bridge = CvBridge()
        self.latest_image = None
        
        # 訂閱相機
        self.create_subscription(
            Image, '/camera/image_raw', 
            self.image_callback, 10
        )
        
        # 建立服務
        self.create_service(
            Trigger, '/capture_snapshot', 
            self.capture_callback
        )
        
        self.get_logger().info('Snapshot service ready')
    
    def image_callback(self, msg):
        self.latest_image = msg
    
    def capture_callback(self, request, response):
        if self.latest_image is None:
            response.success = False
            response.message = 'No image available'
            return response
        
        try:
            # 轉換為 OpenCV 格式
            cv_image = self.bridge.imgmsg_to_cv2(
                self.latest_image, 'bgr8'
            )
            
            # 壓縮為 JPEG (640x480)
            resized = cv2.resize(cv_image, (640, 480))
            _, jpeg = cv2.imencode(
                '.jpg', resized, 
                [cv2.IMWRITE_JPEG_QUALITY, 60]
            )
            
            # Base64 編碼
            b64 = base64.b64encode(jpeg.tobytes()).decode()
            
            response.success = True
            response.message = b64
        except Exception as e:
            response.success = False
            response.message = str(e)
        
        return response

def main():
    rclpy.init()
    node = SnapshotService()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## ⚠️ 注意事項（重要！）

### 1️⃣ snapshot_service 註冊步驟

建檔後**必須**完成以下步驟才能使用：

```bash
# 1. 編輯 setup.py，在 entry_points 加入：
#    'snapshot_service = go2_robot_sdk.snapshot_service:main'

# 2. 重建 package
cd ~/ros2_ws
colcon build --packages-select go2_robot_sdk

# 3. 重新載入環境
source install/setup.zsh

# 4. 測試執行
ros2 run go2_robot_sdk snapshot_service
```

若要用 launch 啟動，需在 `robot.launch.py` 增加該 node。

### 2️⃣ Nav2 / Action 驗證注意

> ⚠️ **使用既有地圖時，必須先設定 Initial Pose！**

- AMCL 需要初始位置才能收斂
- 未設定 Initial Pose 直接送 `/navigate_to_pose` → **Action 會卡住**
- 使用 SLAM 即時建圖時，預設 (0,0) 較少問題

**設定 Initial Pose 方法：**
1. 在 Foxglove / RViz2 用「2D Pose Estimate」工具點擊
2. 或透過 `/initialpose` topic 發布

### 3️⃣ rosbridge 單一實例

> ⚠️ **啟動前務必檢查 Port 9090！**

```bash
# 檢查是否已有 rosbridge 運行
ss -tlnp | grep 9090

# 若有輸出，記下 PID 並決定是否沿用
# 若需重啟，先 kill 舊的
sudo kill <PID>
```

### 4️⃣ 控制安全

- **測試 `/cmd_vel` 時保持低速**（≤ 0.3 m/s）
- 若 **Action 驗證失敗**，立即轉 **Plan B**：
  1. 開發 FSM / Safety Layer
  2. 限制最大速度與加速度
  3. 實作急停功能
  4. 再轉接 Nav2
