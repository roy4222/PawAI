## 🔧 GPU Server 連線問題排除

### 問題 1：SSH 連線 Timeout

**症狀：**
```bash
ssh -L 8050:localhost:8050 -p 8022 roy422@140.136.155.5
# ssh: connect to host 140.136.155.5 port 8022: Connection timed out
```

**原因：** 沒有使用 SSH Config 中定義的 Host 別名

**解法：** 使用 Host 別名連線
```bash
ssh -L 8050:localhost:8050 GPUServer
```

**SSH Config 設定（`~/.ssh/config`）：**
```
Host GPUServer
    HostName 140.136.155.5
    User roy422
    Port 8022
```

---

### 問題 2：SSH Tunnel 建立後 API 仍無法連線

**症狀：**
```
channel 3: open failed: connect failed: Connection refused
```

**原因：** GPU Server 上的 Perception Server 未啟動

**解法：** 登入 GPU Server 後啟動服務
```bash
conda activate torch_env
cd ~/Yoloworld
python perception_server_v2.py
```

---

### 問題 3：Perception Server 缺少依賴

**症狀：**
```
ModuleNotFoundError: No module named 'fastapi'
ModuleNotFoundError: No module named 'cv2'
```

**解法：** 安裝缺少的套件
```bash
pip install fastapi uvicorn python-multipart opencv-python ultralytics supervision
```

---

### 問題 4：Port 不匹配

**症狀：** SSH Tunnel 轉發 8050，但 Server 跑在 8001

**解法：** 重新建立正確 Port 的 Tunnel
```bash
# 關掉舊 Tunnel，重新執行（允許外部連線）
ssh -L 8001:localhost:8001 -L 0.0.0.0:8001:localhost:8001 GPUServer
```

---

### 問題 5：Mac VM 無法連線 Windows Tunnel

**症狀：** Mac VM 用 `localhost:8001` 無法連線

**原因：** SSH Tunnel 在 Windows 上，Mac VM 要連 Windows IP

**解法：** 使用 Windows IP
```bash
# 從 Mac VM 連線
curl http://192.168.1.146:8001/
```
