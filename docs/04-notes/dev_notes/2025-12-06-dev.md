## ⚠️ 重要：VM 網路環境說明（來自 2025/12/01 除錯經驗）

你的 VM 有**雙網卡**，必須正確配置 CycloneDDS：

| 網卡 | IP | 用途 |
|------|-----|------|
| `lo` | 127.0.0.1 | **本機 ROS2 通訊（必須用這個！）** |
| `enp0s1` | 192.168.12.222 | 連接 Go2 機器狗 |
| `enp0s2` | 192.168.1.125 | 連接家用網路（Windows 在這裡） |

**關鍵設定：** 使用 `/home/roy422/local_only_v2.xml` 強制本機通訊走 loopback！

```bash
# 每個 Terminal 都必須設定！
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
export CYCLONEDDS_URI=/home/roy422/local_only_v2.xml
```

## 🎯 今日任務清單

### Phase 1：VM 端環境建置 (30 分鐘)

- [ ] **1.1 安裝 rosbridge_server**
  ```bash
  sudo apt install ros-humble-rosbridge-server
  ```

- [ ] **1.2 確認 CycloneDDS 配置存在**
  ```bash
  cat /home/roy422/local_only_v2.xml
  # 應該包含 <NetworkInterface name="lo"/>
  ```

- [ ] **1.3 啟動 rosbridge（需正確環境變數！）**
  ```bash
  source /opt/ros/humble/setup.zsh
  export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
  export CYCLONEDDS_URI=/home/roy422/local_only_v2.xml
  
  ros2 launch rosbridge_server rosbridge_websocket_launch.xml
  ```

- [ ] **1.4 確認 rosbridge 啟動成功**
  ```bash
  # 另開一個 Terminal
  ss -tlnp | grep 9090
  # 應該看到 rosbridge 在監聽
  ```

### Phase 2：連線測試 (30 分鐘)

> ⚠️ **rosbridge 預設監聽 0.0.0.0:9090**，可從任何網卡存取

- [ ] **2.1 本機測試 WebSocket**
  ```bash
  # 安裝 websocat (如果沒有)
  sudo apt install websocat
  
  # 測試本機連線
  websocat ws://127.0.0.1:9090
  ```

- [ ] **2.2 從 Windows 測試連線**
  ```bash
  # Windows PowerShell (wscat)
  wscat -c ws://192.168.1.125:9090
  ```

- [ ] **2.3 測試 rosapi 服務**
  在連線中輸入：
  ```json
  {"op": "call_service", "service": "/rosapi/topics"}
  ```
  應該回傳 Topic 列表

## ⚠️ 已知問題與解決方案

| 問題 | 原因 | 解決方案 |
|------|------|---------|
| **Terminal 間看不到 topics** | CycloneDDS 走錯網卡 | 確保所有 Terminal 都設定 `CYCLONEDDS_URI=/home/roy422/local_only_v2.xml` |
| **ros2 node list 可看到，但 topic hz 無數據** | Discovery 成功但 Data Transfer 失敗 | 使用 loopback 配置 |
| **Port 9090 無法從 Windows 存取** | 防火牆或網路配置 | 確認 enp0s2 (192.168.1.125) 可 ping 通 |
| **uvx 找不到** | 未載入環境 | `source ~/.bashrc` 或重開 Terminal |
