# 人臉辨識待研究模型與方向

> 目標：把目前「可跑的 CV baseline」升級成「可穩定多人辨識、可除錯、可擴充到 Go2 互動」的版本。

## 研究優先順序

## P0（先做，直接影響可用性）

1. 偵測模型升級（取代 Haar）
   - 候選：YuNet、YOLOv8n-face、RetinaFace（輕量 backbone）
   - 驗證重點：
     - 小臉/側臉召回
     - 720p 單路推論 FPS
     - ONNX/TensorRT 可轉換性

2. 辨識模型升級（取代灰階 flatten 特徵）
   - 候選：SFace（OpenCV FaceRecognizerSF）、ArcFace（InsightFace）、MobileFaceNet
   - 驗證重點：
     - 多人誤認率（FAR）
     - 真人被拒率（FRR）
     - 單幀 embedding 延遲

3. 閾值校準與穩定化策略
   - 方向：
     - upper/lower hysteresis
     - stable hits
     - unknown grace
   - 驗證重點：
     - 避免 `known/unknown` 抖動
     - 多人同框時 label 穩定度

## P1（第二波，提升可靠性）

4. Depth 融合 anti-spoof（D435）
   - 方向：
     - 深度有效像素比例
     - ROI depth median/variance
     - landmark 深度幾何差
   - 驗證重點：
     - 照片/螢幕重放攔截率
     - 真人誤拒率不可過高

5. 多人追蹤與身份綁定
   - 候選：SORT、ByteTrack（先輕量）
   - 驗證重點：
     - track_id 穩定
     - 身份與 track 維持一致

6. Web 可觀測性與除錯體驗
   - 方向：Next.js + FastAPI
   - 驗證重點：
     - 按鈕操作回饋（start/stop/toast）
     - stream health 狀態燈
     - enroll/infer logs 即時可讀

## P2（第三波，提升上限）

7. 雲端辨識路線（RTX）
   - 方向：Jetson 做偵測/裁切，雲端做 embedding + 向量比對（Faiss/Qdrant）
   - 驗證重點：
     - 端到端延遲
     - 網路抖動容錯
     - unknown 決策穩定

8. 頭部朝向/視線估計
   - 候選：6DRepNet、OpenFace
   - 驗證重點：
     - 角度估計穩定
     - 與互動策略整合成本

## 現階段建議模型組合（可落地）

### 低風險快速版
- 偵測：YuNet
- 辨識：SFace
- 防偽：Depth rule-based gating
- 優點：工程成本低、可快速取代 baseline

### 中風險高上限版
- 偵測：YOLOv8n-face 或 RetinaFace
- 辨識：ArcFace（InsightFace）
- 比對：Faiss/Qdrant
- 優點：多人與複雜場景精度上限更高

## 研究與驗收指標（最少要記錄）

- Detection Recall（目標 >= 95%）
- FAR / FRR / EER（需有場域校準）
- 端到端延遲（P95、P99）
- FPS（平均與最小值）
- 長時間穩定性（24h 以上）
- 重啟恢復時間（服務中斷後恢復）

## 下一步執行建議

1. 先跑「YuNet + SFace」替換現有 CV baseline，取得第一版可比較數據。
2. 同步建立 FAR/FRR 測試資料流程（多人、不同角度與距離）。
3. 若誤認仍高，再切到 ArcFace 路線並保留現有穩定化決策層。
